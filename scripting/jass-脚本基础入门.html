JASS 脚本基础入门 ​

JASS简介与学习准备 ​

本节将带你认识JASS脚本语言，了解为什么学习它对你的地图创作有帮助，并学会如何打开JASS编辑环境。完成本节后，你将能够顺利进入JASS代码编写界面，为后续学习打下基础。

什么是JASS脚本 ​

JASS是"Just Another Scripting Syntax"的缩写，中文可以理解为"另一种脚本语法"[^7]。它是一种专门为《魔兽争霸3》设计的编程语言，用于控制游戏中的各种行为，比如单位的移动、攻击、创建，以及游戏逻辑的判断和执行[^6]。

💡 新手提示：把JASS想象成一本"游戏控制说明书"。你用JASS写下指令，告诉电脑在什么情况下该做什么事情。比如"当英雄死亡时，播放音效并显示文字"这样的规则。

为什么学习JASS ​

在《魔兽争霸3》中，你可能已经用过"触发器编辑器"（Trigger Editor）来创建游戏规则。触发器编辑器提供了可视化界面，让你可以通过点击和选择来创建规则。但实际上，每一你在触发器编辑器中创建的操作，背后都会自动生成对应的JASS代码[^4]。

学习JASS有以下几个重要理由：

  1. 更精确的控制 — 有些游戏功能只能在JASS中实现，可视化界面无法完成[^1]
  2. 理解底层原理 — 知道代码在做什么，能帮你更好地调试和优化地图
  3. 阅读和修改 — 你可以查看暴雪自带触发器生成的代码，学习高级技巧

⚠️ 常见错误：很多新手觉得"我不用学代码，用可视化界面就够了"。但当你遇到复杂需求时，比如自定义技能系统或特殊游戏机制，就会发现JASS知识是必须掌握的核心技能。

打开JASS编辑环境 ​

JASS编辑器并不是一个独立的程序，它集成在世界编辑器（World Editor）内部的触发器编辑器中。下面我们一步步来打开它：

  1. 第一步：启动世界编辑器 — 找到并打开《魔兽争霸3》的安装目录，双击"World Editor.exe"。如果桌面上有快捷方式，也可以直接双击打开[^2]

  2. 第二步：打开触发器编辑器 — 在世界编辑器主界面，找到顶部菜单栏中的"触发器"按钮（图标像一个闪电符号），点击它即可打开触发器编辑器窗口

  3. 第三步：进入JASS代码编辑模式 — 在触发器编辑器中，选择菜单 编辑 → 新建自定义脚本，或者直接按快捷键 Ctrl + U。这将创建一个新的空白JASS脚本文件

💡 新手提示：首次打开触发器编辑器时，会看到一些系统自带的预设触发器（如"Map Initialization"）。你可以双击打开它们查看生成的JASS代码，这是学习JASS的好方法！

小结 ​

完成以上步骤后，你应该已经：

  • 理解了JASS是什么（游戏脚本语言）
  • 明白了学习JASS的价值（更强大的地图控制能力）
  • 能够打开JASS编辑环境，进入触发器编辑器并创建自定义脚本

接下来，我们将开始学习JASS的基础语法和常用命令。准备好了吗？

JASS基本语法结构 ​

本节将带你认识JASS代码的"外貌"——了解代码长什么样、怎么写注释、以及最容易出错的书写规范。学完之后，你就能够读懂最简单的JASS代码片段了。

代码基本结构 ​

JASS代码看起来复杂，但它的基本结构其实很有规律，就像搭积木一样。每一块积木（叫做"函数"）负责完成一个具体任务。

  1. 首先，打开触发器编辑器 — 在World Editor顶部菜单栏点击"触发器"按钮[^1]
  2. 创建一个新的触发器 — 右键触发器列表，选择"新建触发器"，双击打开编辑界面
  3. 点击"编辑"菜单 — 在触发器编辑界面找到并点击"编辑"→"转换为JASS"[^2]
  4. 观察代码窗口 — 你会看到类似这样的结构：
jass
function MyFunction takes nothing returns nothing
    // 这里是函数的代码
endfunction
1
2
3

每一段JASS代码都以 function 开头，以 endfunction 结束，中间的 takes 后面写这个函数需要什么参数，returns 后面写它会输出什么结果。

💡 新手提示：记住 function 和 endfunction 必须成对出现！漏掉任何一个都会报错。建议写代码时先把这对单词写好，再往中间填内容。

注释的使用方法 ​

注释就像是代码的"小抄"——你可以在代码旁边写中文说明，让自己以后阅读时能快速想起这段代码是做什么的。计算机不会执行注释内容，所以你可以放心写中文[^4]。

  1. 单行注释 — 在代码前面加 //，这行后面的内容就会变成灰色，不会被程序执行
jass
function Trig_Example_Actions takes nothing returns nothing
    // 这是一个注释，计算不会管它
    call DisplayTextToForce(GetPlayersAll(), "你好") // 这里的注释在代码后面
endfunction

jass
/*
这段注释可以写很多很多行
非常适合写详细的说明
比如解释整个触发器的功能

⚠️ 常见错误：新手经常把注释符号写反，比如写成 //*/。正确的单行注释只有一个 //，正确的多行注释是 /* 开头、*/ 结尾。

大小写与空格规范 ​

JASS对大小写非常敏感！这是个很容易踩的坑。DisplayTextToForce 和 displaytexttoforce 在JASS眼里是完全不同的两个东西[^5]。

  1. 严格区分大小写 — JASS中所有函数名、关键字、变量名都必须使用正确的大小写[^6]
  2. 推荐习惯 — 函数名通常首字母大写，其他单词首字母也大写（如 GetTriggerUnit、CreateUnitAtLoc）
  3. 空格的使用 — 关键字之间、参数之间需要用空格隔开，但不要在单词中间加空格

💡 新手提示：最好的学习方法是在World Editor中直接复制粘贴官方示例代码，而不是自己手动输入。手动输入容易出错，而复制来的代码大小写一定是正确的。

小结 ​

完成以上内容后，你应该：

  • 能认出JASS代码的 function...endfunction 基本框架
  • 能在代码中写单行注释 // 和多行注释 /* */
  • 记住JASS区分大小写，函数名不能写错
  • 下一步可以学习如何用JASS写一个简单的触发器逻辑了！

变量与数据类型 ​

本节将带你认识 JASS 脚本中最基础也最重要的概念——变量与数据类型。学完本节后，你将能够创建自己的变量，并在脚本中正确使用不同类型的数据。

声明变量 ​

在 JASS 中，声明变量就像给一个"盒子"贴上标签一样，你需要给这个盒子起个名字，并告诉计算机里面要装什么类型的东西。

操作步骤：

  1. 理解变量声明的语法 — 在 JASS 中，我们使用 set 关键字来声明变量[^1]。语法格式是：set 变量名 = 值[^4]

  2. 创建第一个变量 — 例如，如果你想创建一个叫 hp 的整数变量，可以这样写：

    jass
    set udg_hp = 100
    1

    这里的 udg_ 前缀表示这是一个全局变量（后面会详细讲解）

  3. 注意变量名的命名规则 — 变量名必须以字母开头，可以包含数字和下划线，不能使用空格或中文[^5]

⚠️ 常见错误：新手经常忘记在变量名加 udg_ 前缀，导致变量无法被触发器识别。一定要记住，全局变量必须加 udg_ 前缀！

常见数据类型 ​

数据类型决定了变量可以存放什么种类的信息。就像不同的盒子适合装不同的东西一样，不同的数据类型也有各自的用途[^6]。

常见数据类型一览：

数据类型中文名用途示例
integer整数存放整数（如 1、100、-5）set udg_score = 999
real实数存放小数（如 3.14、0.5）set udg_damage = 50.5
string字符串存放文字set udg_name = "Hero"
boolean布尔值存放真假（true/false）set udg_isAlive = true
unit单位存放游戏中的单位set udg_hero = gg_unit_H000_0001

💡 新手提示：在 WE 的变量编辑器中创建变量时，系统会要求你选择数据类型。选错类型会导致脚本运行错误，所以创建前要先想清楚这个变量要存什么！

全局变量与局部变量 ​

理解全局变量和局部变量的区别，是写出正确脚本的关键一步[^2]。

全局变量：

  • 在地图加载时就存在，整个脚本运行期间都可以使用
  • 在变量编辑器中创建的变量都是全局变量
  • 使用时需要在名字前加 udg_ 前缀（意为 "User-Derined Global"）[^5]

局部变量：

  • 在函数内部声明，只在该函数执行时存在
  • 使用 local 关键字声明，例如：local integer count = 0[^4]
  • 函数执行完毕后，局部变量会被自动销毁

何时使用哪种？

场景推荐使用
需要在多个触发器间共享数据全局变量
只在一个函数内部临时使用局部变量
需要保存玩家的选择或进度全局变量

💡 新手提示：作为新手，建议先只使用全局变量。虽然局部变量效率更高，但对于 KK 平台的基础地图来说，全局变量已经完全够用，而且更容易管理！

小结 ​

完成以上学习后，你应该已经掌握了：

  • ✅ 如何使用 set udg_变量名 = 值 的格式声明全局变量
  • ✅ 认识了 integer、real、string、boolean、unit 等常用数据类型
  • ✅ 理解了全局变量与局部变量的区别

动手试试看：

  1. 打开 WE 的变量编辑器（触发器编辑器上方有一个📦图标）
  2. 创建一个名为 PlayerGold 的整数变量
  3. 在触发器中使用 set udg_PlayerGold = 500 给玩家初始500金币

这样，你就完成了第一个属于自己的变量操作！

函数基础 ​

本节我们将学习JASS中的"函数"概念。函数就像是厨房里的"菜谱"——你可以提前写好制作步骤（定义函数），然后每次需要时调用这个菜谱来执行。掌握函数后，你的脚本会更简洁、更易维护。

如何定义函数 ​

在JASS中，定义一个函数需要使用 function 关键字[^5]。

操作步骤：

  1. 打开触发编辑器 — 在World Editor中按 F4 打开触发编辑器[^1]
  2. 切换到自定义脚本 — 右键点击触发器，选择"编辑" → "自定义脚本"，这允许你直接输入JASS代码[^2]
  3. 输入函数定义 — 按照以下格式书写：
    jass
    function MyFunction takes nothing returns nothing
    // 在这里写你的代码
endfunction
    1
    2
    3
  4. 解释语法：
    • function — 告诉电脑"我要定义一个新函数"
    • MyFunction — 函数的名字（可以自己起，但要用英文和数字）
    • takes nothing — 函数不需要任何"材料"（参数）
    • returns nothing — 函数执行完不返回任何结果

⚠️ 常见错误：新手经常忘记写 endfunction，导致代码报错。每个 function 必须有一个对应的 endfunction，就像每句话要有句号一样。

调用函数 ​

定义好函数后，你可以在任何地方"调用"它——也就是让函数执行它的代码[^6]。

操作步骤：

  1. 在触发器中调用 — 在触发器的"动作"中找到"自定义脚本"选项[^1]
  2. 输入调用语句 — 只需写函数名加括号：
    jass
    call MyFunction()
    1
  3. 理解 call 的作用 — call 是"调用"的意思，它告诉电脑："去执行名为MyFunction的函数"

💡 新手提示：调用函数时，函数必须是在你调用之前已经定义好的。就像你要做菜，得先把菜谱写好，才能按菜谱做菜。

常用原生函数 ​

原生函数是World Editor提前为你准备好的"工具函数"[^4]，你不需要定义它们，直接调用即可。

最常用的几个：

函数名作用示例
BJDebugMsg("消息")在屏幕上方显示调试信息call BJDebugMsg("Hello World")
GetTriggerUnit()获取触发事件的单位常用于单位类触发器
GetOwningPlayer(unit)获取单位所属玩家GetOwningPlayer(GetTriggerUnit())

💡 新手提示：函数名后面跟的括号里的内容叫做"参数"，不同函数需要不同的参数。鼠标悬停在函数上可以看到它需要什么参数。

小结 ​

完成本节学习后，你应该：

  • ✅ 能用 function...takes...returns...endfunction 的格式定义一个简单函数
  • ✅ 能在触发器中使用 call 函数名() 来调用你定义的函数
  • ✅ 能使用 BJDebugMsg() 在游戏中显示信息来测试你的代码

下一步，你可以尝试写一个简单的函数，让它显示"我的第一个函数！"的消息。

触发器与JASS的结合 ​

本节将教你如何把GUI触发器（图形界面触发器）转换为JASS代码，并帮助你理解在什么情况下应该选择使用JASS而不是纯GUI触发器。完成学习后，你将能够自由切换两种模式，并根据实际需求做出选择。

触发器转JASS方法 ​

  1. 打开触发器编辑器 — 在World Editor中，点击顶部菜单的"触发器编辑器"按钮（也称为Trigger Editor），进入触发器管理界面[^2]。

  2. 选择你要转换的触发器 — 在左侧触发器列表中，找到你想要查看JASS代码的触发器，用鼠标单击选中它。

  3. 切换到JASS视图 — 选中触发器后，点击菜单栏的"编辑"→"转换为JASS"，或者直接按快捷键 Ctrl + Alt + C[^4]。World Editor会将该触发器的所有条件（Conditions）和动作（Actions）转换为对应的JASS代码。

  4. 查看并编辑JASS代码 — 转换后，代码会显示在右侧的代码编辑区域。你可以直接在这里修改代码、添加注释，或者复制代码到其他位置使用。

💡 新手提示：转换后的JASS代码通常包含很多 BJ 开头的函数（如 GetTriggerUnit 对应 GetBJ...），这些是暴雪提供的"便利函数"，初学者可以直接使用它们，不需要完全理解底层原理。

⚠️ 常见错误：很多新手转换后发现代码看不懂，就直接删除整个触发器重新来过。正确的做法是先仔细对比GUI触发器的设置和JASS代码的对应关系——每一行条件/动作都有对应的代码行，理解对应关系才是学习JASS的正确起点。

建议使用JASS的场景：

  1. 需要创建自定义函数 — 当你在多个触发器中重复使用相同的逻辑时，可以将这些逻辑写成JASS自定义函数，避免代码冗余[^1]。

  2. 需要更精细的事件响应 — JASS支持更多类型的事件（Event），比如单位属性变化、游戏时间精确控制等[^6]。

  3. 性能敏感的操作 — 对于需要大量循环计算或实时检测的任务，直接用JASS编写往往比GUI触发器更高效。

  4. 使用第三方JASS库 — 许多社区提供的脚本库（如TESH高亮支持库、自定义伤害系统）需要以JASS形式加载[^1]。

仍建议使用GUI触发器的场景：

  • 地图逻辑相对简单，没有复杂的循环或数学计算
  • 你还不熟悉JASS语法，不想在调试上花费过多时间
  • 需要快速原型验证创意的阶段

实用技巧与常见错误 ​

本节会介绍编写JASS脚本时新手最常犯的错误，以及让代码更简洁、更高效的实用技巧。学完本节后，你将能够识别并避免常见陷阱，写出更专业的脚本。

常见错误与解决方法 ​

  1. 大小写敏感问题 — JASS是一种对大小写敏感的语言，"SetUnitX"和"setunitx"在JASS眼中是完全不同的东西[^1]。如果编辑器提示"找不到函数"，首先检查大小写是否完全正确。

  2. 缺少分号或引号 — 每条JASS语句必须以分号（;）结尾，字符串内容必须用双引号包围。比如 call DisplayTextToPlayer(p, 0, 0, "你好") ，忘记任何一个符号都会报错。

  3. GUI变量名与JASS变量名混淆 — 在触发器编辑器里创建的变量，JASS里使用的是完全不同的名字。如果提示"找不到变量"，检查是否在脚本顶部正确声明了变量。

⚠️ 常见错误：复制别人代码后报"函数已存在"错误。解决方法很简单——给函数起一个唯一的名字，或者在脚本开头加一行 function main takes nothing returns nothing 检查是否已有同名函数[^3]。

JASS代码优化建议 ​

  1. 多用局部变量 — 在函数内部用 local 关键字声明的变量比全局变量运行速度更快。比如 local unit u = GetTriggerUnit() ，用完就自动释放。

  2. 避免频繁创建和销毁单位 — 创建一个单位需要消耗大量计算机资源（就像真的在地图上生成一个角色）。建议尽量复用已有的单位，而不是反复创建再删除。

  3. 学会使用vJASS — vJASS是JASS的"升级版"，支持库、模块、结构体等高级功能[^4]。学会它之后，你会发现写复杂系统变得轻松很多。

💡 新手提示：写完代码后养成用 // 添加注释的习惯。比如 // 这是获取触发单位的代码，这样以后回头看自己能快速理解。注释不会影响程序运行，但能让代码维护变得简单。
