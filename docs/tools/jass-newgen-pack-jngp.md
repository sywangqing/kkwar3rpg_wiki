# JASS NewGen Pack (JNGP)

## JASS NewGen Pack 简介

本节将向你介绍 JASS NewGen Pack（简称 JNGP）是什么，以及它与普通魔兽争霸3编辑器有什么区别。学完本节后，你将明白为什么制作复杂 RPG 地图需要用到 JNGP。

### 什么是 JASS NewGen Pack

JASS NewGen Pack 是一款专门为魔兽争霸3地图开发者设计的**增强版地图编辑器**[^1]。简单来说，它就是在暴雪原版 World Editor 基础上捆绑了多个实用工具的大礼包，其中最重要的组件包括：

- **Grimoire（魔典）** — 提供更强大的触发器编辑界面
- **JassHelper 编译器** — 支持 vJASS 高级脚本语法[^3]
- **TESH** — 为脚本代码添加颜色高亮和语法提示
- **PJass** — 脚本语法检查工具

这些组件共同作用，让地图创作者能够制作出更复杂、更强大的 RPG 地图功能。

### 为什么需要使用 JNGP

如果你只是制作简单的塔防或对战地图，普通 World Editor 勉强够用。但当你想要实现以下功能时，JNGP 就变得必不可少：

> **💡 新手提示**：普通触发器只能处理简单的逻辑判断，而 vJASS 可以让你编写类似"当英雄受到攻击时，有30%几率触发反击特效并扣除敌人10点护甲"这类复杂联动效果。

1. **学习 vJASS 脚本语言** — 这是制作高级 RPG 地图的核心技能[^3]
2. **获得更好的代码编辑体验** — TESH 会给你的代码着色，看起来更清晰
3. **避免常见脚本错误** — PJass 能在你写完代码后自动检查语法问题
4. **访问更强大的触发器功能** — Grimoire 提供了原版没有的高级触发器选项

### JNGP 与普通 World Editor 的区别

| 功能 | 普通 World Editor | JNGP |
|------|------------------|------|
| 触发器语法 | 仅支持基础 JASS | 支持 vJASS（更强大）[^3] |
| 代码编辑 | 无颜色高亮 | TESH 自动着色 |
| 兼容性 | 支持 1.26-1.32 所有版本 | 仅支持 1.26-1.27 版本[^5] |
| 附加功能 | 无 | Grimoire 扩展包等 |

> **⚠️ 常见错误**：很多新手下载了最新版的 JNGP 1.5g 后发现无法使用。这是因为 1.5g 版本只支持旧版本魔兽争霸3[^2]。如果你玩的是 1.28 或更新版本，目前只能使用普通 World Editor 或等待民间重制版。建议先确认自己游戏版本再下载对应工具。

### 小结

JASS NewGen Pack 是魔兽争霸3地图制作领域最重要的工具之一[^3]，它让创作者能够使用 vJASS 编写复杂脚本，从而实现普通触发器无法完成的高级功能。作为新手，你应该尽早熟悉这个工具，为后续学习地图脚本编程打下基础。

## JNGP 下载与安装

JNGP（Jass NewGen Pack）是玩转 KK 地图编辑器最重要的武器——它能让你用更强大的 vJASS 语言来编写触发器逻辑。没有它，你只能做很简单的触发器；有了它，你几乎可以做出任何复杂的游戏系统！

> **⚠️ 重要警告**：JNGP 只能用于 **1.27b 及以下版本**的魔兽争霸3！KK 平台目前仍然使用旧版客户端，所以放心下载吧。

### 下载 JNGP 的正确版本

JNGP 有两个主要分支：

1. **官方 JNGP 1.5g** — 包含 Grimoire、JassHelper（编译器）、PJass、TESH 等组件[^1]
2. **Rebuild / Exp2 版本** — 基于 5d 的第三方魔改版，包含更多修复[^2][^4]

**新手推荐下载 1.5g 官方版**，稳定可靠，社区资源最多。

### 安装步骤详解

#### 第一步：下载 JNGP 压缩包

1. 打开浏览器，访问 HIVE 论坛的 JNGP 官方页面[^1]
2. 找到 "Download" 按钮下载压缩包（约 20-30MB）
3. 把压缩包保存到**桌面**或**新建的文件夹**中，方便后续操作

> **💡 新手提示**：建议建立一个专门的文件夹，比如 `D:\WC3工具`，把所有地图制作相关的软件都放在一起。

#### 第二步：解压文件

1. 如果你没有解压软件，先下载 **7-Zip**（免费）
2. 右键点击压缩包 → 选择 "解压到当前文件夹"
3. 解压后，你应该能看到一个包含 `World Editor.exe` 的文件夹

> **⚠️ 常见错误**：很多新手直接把文件解压到桌面，然后双击 `World Editor.exe` 报错了！原因是 JNGP 的编辑器需要在**原版魔兽争霸3的目录下**运行，或者需要**复制到魔兽3安装目录**中。

#### 第三步：安装到魔兽争霸3目录

这是最关键的一步！

1. 找到你的**魔兽争霸3安装目录**（通常在 `C:\Program Files\Warcraft III` 或 `C:\Program Files (x86)\Warcraft III`）
2. **备份原版 World Editor.exe**！复制一份到其他位置
3. 将 JNGP 文件夹中的 `World Editor.exe` 和其他文件**复制到魔兽争霸3安装目录**，覆盖原文件
4. 同样复制 `jasshelper.exe` 和 `grimoire.exe` 到该目录

> **💡 新手提示**：覆盖前务必备份原版编辑器！万一以后不需要 JNGP 了，还能恢复。

#### 第四步：验证安装成功

1. 进入魔兽争霸3安装目录
2. 双击 `World Editor.exe` 启动编辑器
3. 等待几秒钟，你应该能看到**蓝色的高级触发器编辑器界面**（普通编辑器是灰绿色的）
4. 点击菜单栏的 **「模块」→ 确认能看到「vJass」选项** ✓

> **⚠️ 常见错误**：如果双击后没反应或报错，检查是否把文件复制到了**正确的魔兽3目录**。另外，游戏本体必须是 1.27b 或更低的版本。

### 小结

恭喜你完成了 JNGP 的安装！现在你的地图编辑器已经升级为**超级编辑器**，可以：

- ✅ 编写 vJASS 代码
- ✅ 使用更强大的触发器语法
- ✅ 安装各种社区脚本和库

打开编辑器，新建一张地图，试试新建一个「自定义触发器」，看看是否能看到蓝色代码编辑界面。如果可以，说明安装成功，可以开始你的第一个 vJASS 脚本了！

## JNGP 界面与基本功能

本节将带你认识 JNGP（民间称为"新蛋"）的界面布局和常用功能。学完本节后，你将能够顺利打开 JNGP，识别各个窗口区域，并找到编写脚本的入口。这是你踏上 JASS 编程之路的第一步！

### NewGen 编辑器界面布局

JNGP 本质上是在原有魔兽地图编辑器（World Editor）的基础上，集成了一套更强大的脚本工具。打开 JNGP 后，你会看到和普通 WE 几乎一样的界面，但顶部工具栏会多一些紫色或蓝色的小图标。

> **💡 新手提示**：第一次打开 JNGP 时，界面可能看起来和普通 WE 一模一样，别慌！那些增强功能都"藏在"工具栏和触发器编辑器里，你只需要知道在哪里找它们即可。

### 常用工具栏功能说明

1. **第一步：找到增强工具栏** — 在菜单栏下方，找到一排带有紫色/蓝色图标的小按钮，这些就是 JNGP 独有的增强功能按钮[^1]
2. **第二步：识别"Compile"按钮** — 这个按钮通常是一个齿轮或齿轮+三角图标，点击它可以将你写的 vJASS 代码编译成游戏能识别的格式[^3]
3. **第三步：找到"Test Map"按钮** — 这个按钮看起来像一个小绿三角（▶），点击后 JNGP 会自动保存地图并启动魔兽游戏测试

> **⚠️ 常见错误**：很多新手写完脚本后忘记点"Compile"就直接测试地图，结果发现脚本根本没生效。切记：**每次修改 JASS 代码后，都要先点 Compile！**

### JASS 脚本编辑窗口

在普通 WE 的触发器编辑器中，每个触发器底部有一个"编辑"按钮。JNGP 的强大之处在于，点击这个按钮后打开的脚本窗口有了语法高亮功能——关键字会用不同颜色显示，让你更容易阅读和排错[^1]。

1. **第一步：打开一个触发器** — 在触发器列表中双击任意一个触发器
2. **第二步：进入脚本编辑模式** — 点击触发器底部的"编辑"或"Edit Script"按钮
3. **第三步：观察语法高亮** — 你会看到 `function`、`if`、`then`、`endif` 等关键词用不同颜色标出，这就是 TESH 插件在起作用[^1]

### 小结

完成以上步骤后，你应该已经能够：
- ✅ 识别 JNGP 的界面和普通 WE 的区别
- ✅ 找到 Compile 按钮并理解它的作用
- ✅ 打开 JASS 脚本编辑窗口，欣赏语法高亮功能

> **💡 新手提示**：JNGP 只能在魔兽 1.26 或 1.27 版本下正常运行[^5]，如果你用的是 1.28 及以上版本，JNGP 将无法工作。下节课我们会讲到如何解决版本兼容问题！

## JASS 基础语法入门

本节将带你入门 JASS 编程语言（一种类似传统编程脚本的语言）。学完本节后，你将理解 JASS 中如何存储数据（变量）、如何编写可重复使用的代码块（函数），以及如何将你用图形界面创建的触发器转换为真正的 JASS 代码[^1]。

### 操作步骤

1. **第一步：认识 JASS 编辑器界面** — 打开 JNGP（增强版地图编辑器）后，在顶部菜单栏点击「触发器编辑器」(Trigger Editor) 按钮。在左侧你会看到文件夹列表，右侧是代码编辑区域[^1]。

2. **第二步：理解 JASS 变量与数据类型** — 变量就像一个带标签的收纳盒，用来存放数据。在 JASS 中，常见的数据类型包括：
   - `integer`（整数）：存放 1、2、100 这样的数字
   - `real`（实数）：存放 1.5、3.14 这样的小数
   - `string`（字符串）：存放文字，如 "你好"
   - `boolean`（布尔值）：只有 `true`（真）或 `false`（假）
   
   声明变量的语法是 `local 类型 变量名`，例如：`local integer gold = 100`[^3]

3. **第三步：学习函数定义与调用** — 函数是一段可以反复使用的代码。定义函数使用 `function 函数名 takes 参数类型 参数名 returns 返回类型` 开始，以 `endfunction` 结束。例如：
   ```jass
   function AddGold takes unit u, integer amount returns nothing
       call SetUnitState(u, UNIT_STATE_LIFE, GetUnitState(u, UNIT_STATE_LIFE) + amount)
   endfunction
   ```
   调用函数只需写函数名和参数：`call AddGold(hero, 50)`[^3]

4. **第四步：将触发器转换为 JASS 代码** — 在标准触发器编辑器中右键点击任意触发器，选择「转换为 JASS」(Convert to Custom Text)。这会将图形化的触发器显示为纯 JASS 代码[^1]。这是学习 JASS 语法的绝佳方法——看看系统生成的代码长什么样！

> **💡 新手提示**：JASS 代码区分大小写！`SetUnitLife` 和 `setunitlife` 是完全不同的东西。写错大小写会导致程序报错。

> **⚠️ 常见错误**：新手经常忘记在函数末尾加 `endfunction`，或者在 `call` 语句后忘记加括号传递参数。写完代码后按 F6 检查语法，红色高亮的行就是出错的地方。

### 小结

完成以上步骤后，你应该能够：
- 在 JASS 编辑器中阅读和理解简单的代码
- 知道变量有哪些类型以及如何声明
- 理解函数的结构，能读懂别人写的函数
- 会使用触发器的「转换为 JASS」功能来学习官方代码的写法

⚠️ **重要提醒**：JNGP 仅适用于 Warcraft III 1.26 和 1.27 版本[^5]。如果你使用的是 1.28 或更高版本，JNGP 将无法正常工作[^2]。请先确认你的游戏版本！

## 常见问题与解决方法

在本节中，你将学会如何解决 JASS NewGen Pack (JNGP) 安装后无法启动的问题，以及当脚本出现编译错误时如何排查和修复。掌握这些技能后，你就能正常使用这个强大的地图编辑器了。

### 安装后无法启动

1. **第一步：检查游戏版本** — 打开魔兽争霸 III，查看右下角的版本号。JNGP 5d 及更早版本**仅支持 1.26 和 1.27 版本**[^5]。如果你的游戏已经升级到 1.28 或更高版本，编辑器将无法正常启动[^2]。这是最常见的原因！

2. **第二步：确认补丁版本** — 如果版本不对，你需要将游戏降级到 1.27。可以在游戏启动器中选择旧版本，或者从可信渠道下载 1.27 版本的魔兽争霸 III。

3. **第三步：以管理员身份运行** — 右键点击 JNGP 的启动程序，选择"以管理员身份运行"。这样可以避免权限问题导致的启动失败。

4. **第四步：检查杀毒软件** — 有些杀毒软件会误报 JNGP 为危险程序。暂时关闭杀毒软件或将其加入白名单，然后再尝试启动。

> **⚠️ 常见错误**：很多新手遇到"程序无响应"就直接重装，其实 90% 的情况是**游戏版本不匹配**。请一定先确认版本！

### 脚本编译错误排查

1. **第一步：查看错误提示** — 在 JNGP 中点击"编译脚本"后，如果出错，底部会显示红色文字。仔细阅读错误信息，通常会告诉你出错的行号和原因。

2. **第二步：检查语法** — JASS 对语法要求严格，常见错误包括：缺少分号`;`、括号不匹配、变量未声明就使用等。例如，`call PrintMessage("Hello")` 少了结尾的分号就会报错。

3. **第三步：确认 vJASS 标记** — 如果你使用了 vJASS 的高级功能（如结构体、库），必须用 `//! javascription` 标记开始，否则系统会当作普通 JASS 处理导致错误[^3]。

4. **第四步：使用 JassHelper 检查** — JNGP 内置的 JassHelper 会自动检查语法。点击工具栏的"验证"按钮，可以快速定位问题所在。

> **💡 新手提示**：写完脚本后养成好习惯，**每写完一个函数就立即测试编译**，不要等所有代码写完才编译，这样能更快定位错误！

### 小结

完成以上步骤后，你应该能够：
- ✅ 确认自己的游戏版本是否与 JNGP 兼容
- ✅ 成功启动 JASS NewGen Pack 编辑器
- ✅ 读懂脚本编译错误信息并快速定位问题
- ✅ 修复常见的语法错误和配置问题

遇到问题时不要慌，90% 的情况都是版本问题或小语法错误。按照上面的步骤逐一排查，你一定能解决！
