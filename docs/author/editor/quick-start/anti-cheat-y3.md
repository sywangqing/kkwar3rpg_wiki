# Y3编辑器反作弊概要

Y3 中最主要的作弊手段为通过 CE 进行内容修改。目前客户端有基础的防作弊功能和反作弊回档、存档加密、驱动防护，常规CE 修改无法生效，门槛较高。Y3 作弊目前主要通过 CE 分为以下两类

## 修改内存数据，影响存档上

这个方法主要是通过 CE 找到客户端运行中的内存地址，修改存档相关数值，将作弊后的数据上传至服务器存档

目前驱动保护、编辑器反作弊，直接CE 修改并不会生效，且无attach Y3 进程，需要老的 Windows 版本并通过解析偏移地址定位到真实内存，有比较高的技术门槛

### 建议

针对核心资源，特别是货币类存档，在使用常table 存档的情况下，在独立int 存档槽位进行存档，并使用平台提供的防刷限制每局、每次、每单位时间变更上限、上传频率等。可有效防止存档被直接修改到夸张数值

同时可以配合多种存档组合，譬如：

- 通过一*只增存档**，来记录当前资源的消耗量，结合当前存档余量，来获取总量判断是否存在作弊
- 通过一*只增存档**，来记录当前资源在合理逻辑下（地图、商城、赠送）的获得总量，若该值小于存档当前值，可基本认定为作弊
- 通过一个存档来记录每次用户指定资源的最大值，并记录最大值，将该存档设置内部排行榜，快速排查顶部异常数据；
- 其他一些加密手段，table 非明文存

ORPG 此类道具较多、存档不够用、不方便用平台防刷存档，也可以在地图逻辑中写死上限，虽然也存在被修改，但由于定值不变化，较难被定位
