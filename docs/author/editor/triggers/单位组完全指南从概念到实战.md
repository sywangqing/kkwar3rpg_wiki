# 单位组完全指南：从概念到实战

## 什么是单位组（Unit Group）？

本节将介*单位组（Unit Group*这个概念。单位组是魔兽争地图编辑器中用于**同时管理多个单位**的容器。学会单位组后，你就能对一个区域内的所有敌人、或者一群友*批量执行相同的操*，比如同时造成伤害、同时移动、同时播放特效——这将大大简化你的触发器（trigger）编写

### 单位组的定义与作

**单位*你可以把它想象成一*"篮子"**，你可以把任意多个单放进，然后一次性对篮子里的所有单位做操作。比如，你想让地图上某个区域内的所有敌方单位受00点伤害，只需要：

1. 把这些单位放入一个单位组
2. 对这个单位组执行"对每个单位循的操
3. 在循环内部，对每个单位造成100点伤

这样你不需要写10条0条单独的伤害触发，而是用一条简洁的触发搞定所有单位

> **💡 新手提示**：单位组最强大的地方在于它*动态的**——你可以随时向组里添加新单位，或者移除不需要的单位，而不需要事先规这个组只能装5个单

### 单位组与单位数组的区

很多新手会混*单位组（Unit Group**单位数组（Unit Array*，它们都能存储多个单位，但本质不同：

| | 单位| 单位数组 |
|---|---|---|
| **结构** | 无序单位集合" | 有编号的"单位列表" |
| **大小** | 随时可变 | 需要预设固定大|
| **查找** | 无法直接按编号访问第N个单| 可以直接访问 `单位数组[3]` |
| **适用场景** | 一群单批量操作 | 按顺按编号处理单|

简单来说：**单位数组**像是一排编好号的储物柜，你需要知道编号才能取出东西；**单位*像是一个随用随取的篮子，你只需要知我要对篮子里的所有东西做某件

> **⚠️ 常见错误**：新手经常把单位组当成数组来用，试图单位组[1]"的方式访问第1个单位。这在单位组*无法直接实现**——你需要用"对每个单位执行动循环来依次处理组内的单位

### 小结

- 单位组是一*存放多个单位的容*，适合批量操作
- 它与数组的区别在于：**数组用编号管理，单位组用集合管理**
- 后续教程中，你将学习如何**创建、添加、移*单位组中的单位，以及如何**遍历**组内的单位执行操

## 创建单位组的基本方法

本节将教你三种创建单位组的方法，从最简单的手动创建开始，到使用条件自动筛选。学完本节后，你将能够根据不同需求灵活选择最合适的建组方式

### 手动创建空单位组

这是最基础的建组方法，适合需要手动往组里添加单位的场景

1. **打开触发器编辑器** 在顶部菜单栏点击"触发按钮（或按F4键）[^3]
2. **创建新变* 右键点击左侧变量列表，选择"新建变量"
3. **设置变量类型** 将变量名改为你喜欢的名字（如"我的单位），类型下拉菜单中选择"单位[^3]
4. **初始化空* 在触发器动作中添设置变量"动作，选择你刚创建的变量，等号右边选择"单位- 空单位组"

> **💡 新手提示**：单位组就像一篮子"，你创建的是一个空篮子，后续需要往里面放单位。如果不放任何单位就使用这个组，触发器可能会出错或什么都不会做

> **⚠️ 常见错误**：新手经常忘记先初始化变量就直接使用，结果导致游戏报错。每次使用新的单位组变量前，一定要记得先用"空单位组"初始化它

### 根据条件自动筛选单

这种方法可以自动找出符合特定条件的单位，适合需要对一大群单位进行操作的场景

1. **创建条件筛选触发器** 新建一个触发器，添单位- 选取区域内所有单位满足条动作[^3]
2. **选择筛选区* 点击"区域"输入框，在地图上拖动创建一个你要搜索的范围
3. **设置单位条件** 点击"条件"输入框，在弹出的条件编辑器中选择"单位 - 单位的类等条
4. **保存结果到变* 选取动作执行完毕后，符合条件的单位会自动放入你指定的单位组变量中

> **💡 新手提示**：条件筛选非常强大！比如你可以筛所有血量低0%的骑，然后对他们统一施放治疗法术。筛选条件可以叠加使用，满足多重条件

### 使用预置条件快速建

为了提高效率，编辑器内置了许多常用的预置条件，让你不用每次都手动设置

1. **在选取动作中选择"预置条件"** 展开条件选择器，找到"预置"分类[^4]
2. **选择适合的预* 常用预置包括
   - "存活单位" - 只选取没死的单
   - "同阵营单 - 只选取某一方的单位
   - "可攻击的单位" - 排除建筑、中立单位等
3. **微调预置条件** 某些预置允许你进一步调整参数，比如选择具体是哪个玩家阵

> **⚠️ 常见错误**：新手容易混单位类型"单位标签"。单位类型是骑士"法师"这类分类，而标签是一种自定义标记。别选错了！

### 小结

完成以上三种方法的学习后，你应该能够
- 创建空单位组并手动添加单
- 使用条件自动筛选符合条件的单位
- 利用预置条件快速建

在实际制作中*推荐优先使用预置条件**，因为它既快速又不容易出错。只有当预置条件无法满足需求时，才需要手动设置更复杂的自定义条件

## 单位组的核心操作

本节我们将学习对单位组进行增删改查的基本操作。学完后，你能够灵活地往单位组里添加或移除单位，逐个检查组内的单位，并正确地清空不需要的单位组

### 遍历单位组中的每个单

1. **第一步：创建单位组变* 在触发器编辑器的变量列表中，点击"新建变量"（绿色加号按钮），变量类型选择"单位，给变量起个名字比如`临时单位组`[^1]
2. **第二步：使用"选取区域内单动作** 在触发器动作中找选取区域内单（Units in region），这个动作会自动把某个区域内的所有单位放入你的单位组变量中[^3]
3. **第三步：遍历单位** 展开"选取区域内单动作的子选项，选择"执行动作"，然后添加一个循环。常用的组合是：`For each (Integer A) from 1 to (Number of units in 单位，执行动作`[^3]
4. **第四步：对每个单位操* 在循环内部，你可以使用`Set VariableSet tempUnit = (Picked unit)`获取当前正在处理的单位，然后对这个单位进行你想要的操

> **💡 新手提示**：选择"选取区域内单时，记得先在地图编辑器中创建一*区域（Region*，没有区域就没办法告诉电脑去哪个范围找单位

> **⚠️ 常见错误**：新手容易忘选取区域内单是一*动作（Action*，必须放在触发器*执行（Actions*部分，而不是条件（Conditions）部分

### 添加与移除单

1. **第一步：添加单个单位到组** 在动作列表中找到"Add unit to unit group"，将你想要加入的单位（可以是某个具体单位，也可以是触发事件中触发单位"生效单位"）添加进去[^3]
2. **第二步：使用"选取"动作批量添加** 如果你需要添加多个单位，比如所有某种类型的单位，使选取单位"动作配合"添加到单位组"条件来批量操
3. **第三步：从单位组移除单位** 找到"Remove unit from unit group"动作，选择要移除的单位和你要操作的单位
4. **第四步：特别注意** 移除单位并不会消灭这个单位本身，它只是从组里消失而已。如果你想同时删除单位，需要额外使Kill unit"Remove unit"动作[^3]

> **💡 新手提示**：如果单位被移出单位组后你还想继续使用它，一定要先用一*单位变量**保存它，否则之后就没办法再找到它了

### 清空与销毁单位组

1. **第一步：清空单位* 找到"Clear unit group"动作，选择你要清空的单位组。这个动作会把组*所有单位都移除**，但不会删除那些单位本身[^3]
2. **第二步：销毁单位组（高级）** 如果你使用的*JASS或Lua脚本**，可以用`DestroyGroup()`函数完全销毁单位组释放内存。但在GUI界面中，频繁清空再重新填充通常就够
3. **第三步：判断单位组是否为* 使用"Number of units in unit group"函数检查，如果返回值是0，说明组已经被清空了

> **⚠️ 常见错误**：很多新手混淆了"清空单位删除单位"。清空只是把单位从组里拿出来，单位还在地图上；删除单位（Kill/Remove）才是真正让单位从游戏中消失

### 小结

完成以上三个部分的学习后，你应该掌握了：
- 如何使用"选取区域内单并配合循环遍历组内每个单位进行操
- 如何通过"添加/移除单位到单位组"来动态管理组内成
- 如何使用"清空单位正确地重置一个不再需要的单位

**实战练习**：试着创建一个触发器，让你在地图中央区域放置5个步兵，然后写一个触发器把这些步兵全部选出来并对每个步兵播放一段文字提示。这样你就完成了一次完整的"添加→选取→操的循环练习！

## 单位组的实战应用场景

在了解了单位组的基本概念后，这一节我们将学习**如何在实际地图中运用单位*。学完本节后，你将能够实对一片区域内的所有敌人释放技自动识别战场上的敌我单位"等常见功能

### 群体技能释

群体技能是RPG地图中最常用的功能之一，比对半00范围内的所有敌人造成伤害"。这需要先收集区域内的单位，再对每个单位执行动作

#### 操作步骤

1. **创建触发* 在触发器编辑器中新建一个触发器，命名为"群体AOE技[^1]
2. **设置触发条件** 添加"单位使用技事件，选择你想要触发的技能ID[^2]
3. **创建单位组变* 在变量编辑器中新单位类型的变量，命名为`aoe_targets`[^3]
4. **收集目标单位** 使用"单位- 选取范围内所有单动作，以技能施放点为中心，半径设为300，存入`aoe_targets`变量[^1]
5. **循环遍历单位** 添加"单位- 选取每个单位"动作，对`aoe_targets`中的每个单位施加伤害或附加效果[^3]
6. **清理内存** 动作执行完毕后，使用"单位- 移除单位"销毁临时单位组，避免内存泄漏[^4]

> **💡 新手提示**：群体技能的范围最好设置为可调节的变量，这样你可以轻松修改技能的覆盖半径，而不用改代码

> **⚠️ 常见错误**：新手经常忘记第6步清理单位组。临时创建的单位组如果不销毁，会导致地图运行越来越卡，甚至崩溃

### 区域判定与敌我识

在防守类或塔防类地图中，你可能需要检测某片区域是否被敌人入侵，或者统计当前有多少友军和敌军

#### 操作步骤

1. **定义区域** 在地图编辑器中，区域工具"在需要监测的位置画一个矩形区域[^5]
2. **创建单位* 新建两个单位组变量：`enemies_in_zone`（区域内的敌人）和`allies_in_zone`（区域内的友军）
3. **获取区域内的单位** 使用"单位- 选取区域内的单位"动作，把所有单位放入一个临时组[^1]
4. **过滤敌我** 使用"条件 - 单位属于玩家"单位 - 获取单位所有来区分敌我[^3]
5. **执行对应逻辑** 如果敌人数量大于0，则触发防御机制；如果是友军，则可以触发增益效果

> **💡 新手提示**：敌我识别时，记得考虑"中立敌对"单位（如野怪），它们的所有者通常是玩4（系统）

> **⚠️ 常见错误**：新手容易混玩家1的友玩家1的单。前者包括玩及其盟友的所有单位，后者只包括玩家1自己的单位

### 编队管理与批量操

在rpg地图中，玩家经常需要控制多个单位进行战斗。单位组可以帮助你实一键选中所有前锋部让所有治疗单位治疗血量最低的队友"

#### 操作步骤

1. **建立编队单位* 在触发器中预设一个包含编队成员的单位组，例如`tank_group`包含所有坦克[^1]
2. **响应快捷* 使用"玩家 - 玩家1按下了哪个键"作为触发条件[^3]
3. **批量选中单位** 使用"选择 - 选择/取消选择"动作，对`tank_group`中的所有单位执行选中[^2]
4. **批量下达命令** 使用"单位 - 命令单位移动/攻击"动作，让所有编队单位同时执行指令[^1]

> **💡 新手提示**：批量操作时，可以使单位- 随机选择单位"来让不同单位攻击不同的目标，避免所有单位挤在一起攻击同一个敌人

> **⚠️ 常见错误**：不要在每个帧（游戏循环）内频繁创建和销毁单位组，这会严重影响性能。尽量复用已有的单位组变量

### 小结

通过本节学习，你已经掌握了单位组的三大实战应用：**群体技**敌我识别***编队管理**。这三种场景覆盖了绝大多数RPG地图的核心需求。建议你先从"群体技能释开始练习，因为它最能帮助你理解单位组的"收集→处理→清理"完整流程

## 常见问题与性能优化

本节将帮助你解决使用单位组时最常遇到的"内存泄漏"问题，以及如何在不同场景下选择正确的清空或销毁方式。学完本节后，你的地图将能更稳定运行，即使单位数量很多也不会卡顿

### 单位组泄漏问题解

> **⚠️ 常见错误**：很多新手发现地图运行几分钟后开始变卡，甚至崩溃，这通常*单位组泄*造成的

**单位组泄*是什么？简单说就是：当你创建一个单位组后忘记清理它，这部分"记忆"就会一直占用电脑资源，久而久之电脑就变慢了

#### 操作步骤

1. **理解泄漏的原* 当你使用"设置变量"动作创建一个单位组后，如果不再使用这个变量，WE（World Editor）不会自动清理它[^1]
2. **检查你的触发器** 在触发器编辑器中，找到所有使单位的动作，特别添加单位到单位组"设置单位组变的地
3. **在触发器末尾添加清空动作** 每个可能重复执行的触发器，都应该在最后加清空单位动作，确保每次运行后清理干净[^3]

> **💡 新手提示**：养成习惯——只要触发器里创建了单位组，就一定要在触发器末尾清空它，就像吃完饭要洗碗一样

### 何时清空 vs 何时销

这是两个不同的动作，很多新手搞不清楚

| 情况 | 应该使用 | 原因 |
|------|---------|------|
| 单位组变量还需要用 | **清空** | 只清除内容，变量还在 |
| 完全不再需要这个变| **销* | 完全删除，释放内|

#### 操作步骤

1. **打开触发器编辑器** 双击任意触发器，进入编辑界面
2. **找到单位组动* 在动作列表中，搜清空"销[^3]
3. **清空单位* 选择"清空单位动作，选择对应的变量名。这会移除组内所有单位，但变量本身保
4. **销毁单位组** 选择"销毁单位组"动作。这会完全删除这个单位组，通常用于不再需要的临时

> **💡 新手提示**：一般建*优先使用"清空"**，除非你确定这个变量真的不会再用了

### 优化大型单位组的技

如果你的地图有大量单位（比如怪物群、召唤物），单位组操作会变慢

#### 操作步骤

1. **减少不必要的单位组创* 不要每个X这种循环触发器里频繁创建新单位组，尽量复用已有的变量
2. **使用"选取区域内单时要限定条件** 比如只存活单位"属于某玩，避免选取太多无用单位[^2]
3. **及时清理死亡的单* 使用"单位死亡"事件触发器，在单位死亡时立即将其从相关单位组中移
4. **避免在循环中反复添加/移除单位** 如果必须在循环中操作单位组，先把单位存到一个临时列表或变量里，循环结束后再统一处理

> **⚠️ 常见错误**：新手喜欢在".01的快循环里频繁清空和重建单位组，这会导致严重卡顿。解决方法是把循环间隔改为至.5秒

### 小结

完成以上学习后，你应该：
- 能够识别并修复单位组泄漏问题
- 知道何时使用"清空"、何时使销
- 掌握优化大型单位组的基本技

> **💡 新手提示**：写完触发器后，多测试几遍，观察地图运行时间变长没有。如果变卡了，很可能是某个触发器里有泄漏问题

## 进阶技巧与常见错误

本节将教你如何正确使用嵌套单位组，并帮助你避免新手最常犯的错误。学完本节后，你将能够更高效地管理大量单位，并编写不会导致游戏卡顿的触发器

### 嵌套单位组的正确用法

在复杂地图中，你可能需要将单位按不同类型分组后再统一操作。这时嵌套使用单位组就很有用了

1. **创建主单位组** 在触发器中新建一单位类型的变量（比如所有近战单），获取范围内单动作填充它[^1]

2. **在循环中筛选子* 使用"遍历单位组中每个单位"动作（Pick Every Unit），在循环内部根据条件再创建新的临时单位组[^3]

3. **记得清理临时* 每次使用完临时单位组后，务必清除单位动作释放内存，否则长期运行会拖累游戏性能

> **💡 新手提示**：想象单位组就像一容器"，嵌套使用就是在容器里再放小容器。操作时记得"用完要倒掉"

### 新手常犯的错误与避免方法

> **⚠️ 常见错误 1**：把单个单位和单位组搞混
> - 很多新手会对一个单位使遍历单位动作，这会报
> - 解决方法：单个单位用"单位"类型变量，单位组才用"单位类型变量[^3]

> **⚠️ 常见错误 2**：忘记清除单位组导致游戏变卡
> - 每次获取范围内单创建的单位组，如果不手动清除，内存会一直被占用
> - 解决方法：在触发器结尾添清除单位动作，养成好习惯[^4]

> **⚠️ 常见错误 3**：在循环中重复创建相同的单位
> - 有些新手会在每次循环时都新建单位组，导致效率极低
> - 解决方法：循环外创建一次，循环内反复使用同一个变

### 小结

完成以上步骤后，你应该能
- 正确创建和使用嵌套单位组
- 避免单位组相关的报错
- 编写出不会造成内存泄漏的触发器

记住*单位组是工具，用完要清理，嵌套要谨慎，变量要对齐*
