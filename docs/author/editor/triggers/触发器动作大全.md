# 触发器动作大

## 触发器动作概

触发器动作是让魔兽争地图"动起的核心工具。本节将介绍什么是触发器动作、它们如何分类、以及最常用的动作有哪些。学完本节后，你就能理解触发做什的部分，为后续创建自己的交互逻辑打下基础

### 触发器动作简

触发器由三部分组成：**事件（Event**条件（Condition**动作（Action*。你可以把触发器想象成一如果...那么..."的指令，*动作就是"那么"后面要做的事*。比如果玩家点了一个按钮，那么显示对话框给玩家"—显示对话就是一个动作[^3]

动作是触发器中最灵活的部分。你可以
- 让单位移动或攻击
- 创建或删除单
- 显示文字信息给玩
- 播放音效
- 设置变量保存数据
- 等等...

### 动作的分类与用

触发器动作可以分为以下几大类

| 分类 | 用| 常见动作举例 |
|------|------|-------------|
| **单位动作** | 控制单位的行| 移动单位、创建单位、删除单|
| **玩家动作** | 管理玩家数据 | 设置玩家属性、给玩家加金|
| **对话框动* | 与玩家交| 显示对话框、隐藏对话框 |
| **音效动作** | 播放声音 | 播放音效、等待音效结束[^1] |
| **变量动作** | 存储和读取数| 设置变量[^1]、算数运|

### 常用动作快速索

> **💡 新手提示**：刚开始不用记住所有动作！记住个最常用的就够了
> - **设置变量** 保存数据
> - **如果/那么/否则** 做判
> - **显示文本消息** 反馈给玩

> **⚠️ 常见错误**：新手经常把"动作"条件"搞混。记住：**条件**是判要不要做"*动作**是实做什。比如果金币>=100"是条件，"给玩00金币"才是动作

### 小结

本节我们学习了触发器动作的基本概念：
- 动作是触发器中实际执行操作的步骤
- 动作分为单位、玩家、对话框、音效、变量等多个类别
- 初学者记设置变量、判断逻辑、显示信这三个核心动作就足够入门

下一步，你可以打开World Editor，随便创建一个触发器，试着添加一显示文本消息"动作，看看实际效果！

   - 选择要创建什么单位类型（从单位池中选择
   - 选择创建后属于哪个玩家（就是"拥有
2. **设置单位变量** 刚创建的单位是一对象"，你可以**设置变量** [^1] 把创建出来的单位存到一个变量里，方便后面使
   - 比如：`Set TempUnit = (Last created unit)`
3. **销毁单* 当你需要移除某个单位时，使**单位 - 移除单位** [^2]
   - 只需要把要移除的单位变量拖进去即

> **💡 新手提示**：刚创建的单位变量名`(Last created unit)`，但这个只能立即使用，过一秒就变成别的单位了。一定要第一时间存到自己的变量里

### 单位属性修

#### 操作步骤

1. **找到修改动作** 在动作列表中展开 **单位** 分类，你会看到很多修改选项 [^3]
2. **修改生命* 使用 **单位 - 设置生命** **单位 - 增加生命** [^2]
   - "设置"是直接改成多少，"增加"是在原来的基础上加
3. **修改攻击护甲** 类似的，使用 **单位 - 设置属* 类动
4. **添加buff/技* **单位 - 添加技* 可以让单位学会某个技

> **💡 新手提示**：修改属性时注意单位是否活着——对已经死亡的单位修改属性是无效的！如果你的修改没生效，先检查单位是否还活着

> **⚠️ 常见错误**：新手经常混设置"增加/减少"。比如设置生命为100，如果单位本来有200血，就会变00；如果用增加100，则会变00

#### 操作步骤

1. **找到资源动作** **玩家** 分类下，**玩家 - 设置资源** **玩家 - 增加资源** [^2]
2. **控制金币** 选择对应的玩家，然后操作 **当前* [^3]
   -  مثالية：用"增加资源"来做任务奖励，比"设置"更灵
3. **控制木材** 同理操作 **当前*
4. **广播UI更新** 修改资源后， Warcraft 会自动更新画面显示，但如果你写了自定义界面，可能需要手动刷

> **💡 新手提示**：玩家资源包括金币、木材、食物上限（人口）等。不同游戏模式资源类型可能不同，RPG地图经常还会自定义资源（如积分、钻石）

> **⚠️ 常见错误**：新手用"设置资源"来做奖励，以为会"增加"——结果把玩家的钱全部重置成奖励数额了！如果只多给"，一定要增加"而不设置"

## 游戏流程控制动作

本节将教你如何使用触发器控制游戏生死"——即游戏何时开始、进行多久、什么时候结束，以及根据不同情况做不同反应。学完本节，你就能让地图有了"脑子"，会根据玩家的行为自动执行对应的操作

### 游戏开始与结束

游戏流程控制的基础是知*什么时候该做什么事**。在触发器中Wait"（等待）是最常用的动作之一，它能让触发器暂停执行一段时间，就像你按下计时器等待一样[^1]

1. **打开触发器编辑器** KKWE 顶部菜单点击"触发按钮（像一个闪电图标的按钮
2. **创建一个新触发* 右键点击左侧空白区域，选择"新建触发，给它起名比游戏流程"
3. **添加"等待"动作** 双击触发器，在右侧动作栏找到"通用"分类，双等待"
4. **设置等待时间** 在弹出的数值框中输5.00"，单位是秒——这意味着游戏开始后会等

> **💡 新手提示**等待"后面的数值可以是变量！比如设置一游戏时间"变量，就能动态控制等待多久，这样比写死数字灵活得多

> **⚠️ 常见错误**：很多新手设等待"后发现后面其他动作一起执行了——这是正常的等待"只暂停当前触发器的这一条线，不会暂停其他触发器或游戏本身。如果想让整个游戏暂停，需要用"暂停游戏"动作

### 计时器与循环

当你想让某个动作**重复执行**很多次时，逐个添加"等待"非常麻烦。这时候就要用到循环动作[^2]

1. **找到"整数循环"动作** 在动作列表中展开"通用"分类
2. **双击"For Loop"（整数循环）** 这会自动生成一段循环结
3. **配置循环参数** 第一个框是起始值（），第二个是结束值（），第三个是步长（填1
4. **在循环体内添加动* 把你想重复的动作拖到循环体里

这样设置后，循环体会执行10次（0）。如果你想让循环在某个条件下停止，可以使Exit Loop"（退出循环）动作

### 条件与流程分

这是让地变聪的关键！通过"If-Then-Else"（如那么-否则）动作，触发器可*根据不同情况做不同的*[^3]

1. **添加条件判断动作** 在动作列表找通用"If Then Else"
2. **点击"如果"部分** 会出现一个编辑框，需要在这里设置条件
3. **设置你的条件** 比如"玩家的黄>= 100"，表如果玩家00以上黄金"
4. **那么"分支添加动作** 如果条件成立，执行这些动作（比如扣钱造兵
5. **否则"分支添加动作** 如果条件不成立，执行这些动作（比如提黄金不足"

> **⚠️ 常见错误**：新手经常忘记在条件判断前设否则"分支会发生什么。其否则"是可选的——你可以只设如果"那么"，不否则"部分，触发器就会在条件不满足时什么都不做

### 小结

完成以上学习后，你应该掌握了
- 使用"等待"动作来控制游戏时间的节奏
- 使用"整数循环"让动作重复执
- 使用"If-Then-Else"让地图根据不同情况做出不同反

## 界面与通信动作

本节将教你如何在游戏*向玩家展示信*以及**与玩家进行交*。学完本节后，你能让游戏中的角色与玩对话"，创建弹出对话框让玩家选择，甚至管理任务系统和日志记录

### 文字与消息显

文字显示是最基础的玩家沟通方式，常用于剧情对话、提示信息、调试反馈等场景

1. **第一步：打开触发器编辑器** 在顶部菜单栏点击"触发按钮（像一个闪电符号），进入触发器编辑界面[^1]
2. **第二步：创建新触发器** 右键点击左侧列表，选择"新建触发，命名为"显示消息"
3. **第三步：添加动作** 点击触发器内新增动作"，在动作列表中找游戏-显示文本消息给玩（或"通信"分类下的相关动作）[^3]
4. **第四步：设置参数** 在弹出的窗口中，选择你要显示消息的玩家（所有玩或指定玩家），然后在文本框中输入你想显示的文字，欢迎来到我的RPG地图
5. **第五步：测试运行** 按F6打开游戏测试，观察文字是否正确显示[^5]

> **💡 新手提示**：文字消息默认显示在屏幕左上角，你可以在动作参数中选择不同的显示位置（给玩会让文字显示在对应玩家的屏幕中央附近）

> **⚠️ 常见错误**：新手经常忘记设玩家"参数，导致消息不知道发给谁。务必确认选择了正确的目标玩家

---

### 对话框与按钮

对话框能让玩家做出选择，是RPG游戏中制作菜单、对话分支、NPC交互的核心功能

1. **第一步：创建对话框动* 在触发器动作列表中找对话显示"，或者先使用"对话创建"来创建一个对话框变量[^2]
2. **第二步：设置对话框标* 使用"对话改变标题"动作，为对话框设置一个标题，请选择你的职业"
3. **第三步：添加按钮** 使用"对话添加按钮"动作，添加至少一个按钮。为按钮设置标签文字（如"战士"法师"）[^3]
4. **第四步：为按钮点击创建响应触发器** 新建一个触发器，事件选择"对话点击按钮"
5. **第五步：读取玩家选择** 在响应触发器中，使用"对话获取点击的按动作来判断玩家点击了哪个按钮，然后执行对应的逻辑[^2]

> **💡 新手提示**：创建对话框后，记得使用"对话销动作在玩家做出选择后清理对话框，否则对话框会一直留在屏幕上影响游戏体验

> **⚠️ 常见错误**：新手常忘记为对话框添加事件响应，导致玩家点击按钮后什么都不会发生。每个对话框都必须配套一对话点击按钮"事件触发器

### 任务与日

任务系统是RPG地图的灵魂。通过任务动作，你可以创建任务目标、追踪进度、向玩家日志添加记录

1. **第一步：创建任务** 在动作列表中找到"任务-创建任务"动作，设置任务标题和描述[^3]
2. **第二步：设置任务图标和颜* 使用"任务-改变图标"任务-改变颜色"动作，让任务在日志中更容易辨
3. **第三步：添加任务目标** 使用"任务目标-创建"动作，为任务添加子目标（击败10只怪物"找到隐藏的宝）[^3]
4. **第四步：更新任务进度** 当玩家完成某个目标时，使任务目标-完成"动作标记该目标已完成[^4]
5. **第五步：完成或失败任* 当所有目标达成或任务失败时，使用"任务-完成"任务-失败"动作结束任务，并可设置奖励提示[^1]

> **💡 新手提示**：任务日志默认显示在屏幕右上角，玩家可以随时打开查看当前任务进度。合理利用任务目标能让玩家清楚知道自己该做什么

> **⚠️ 常见错误**：新手往往一次性添加太多任务目标，导致玩家感到困惑。建议每个任务控制在2-4个目标之间，循序渐进

## 特效与音效动

本节将教你如何在地图中添加炫酷的**特效**（如火焰、爆炸光环）*音效**（如技能释放音、背景音乐），以及如何控*摄像*视角，让你的RPG地图不再单调！学完本节，你将能够为技能添加视觉效果、为场景配上背景音乐，并让镜头跟随角色移动

### 视觉效果创建

视觉效果（Special Effect）是指在地图上显示的粒子特效，比如魔法光环、爆炸火焰等。它们能让你的技能看起来更加酷炫。[^1]

#### 操作步骤

1. **打开触发器编辑器** 在顶部菜单栏点击"触发（Trigger）按钮，进入触发器编辑器界面[^2]

2. **创建一个新触发* 右键点击左侧面板，选择"新建触发，命名为"播放火焰特效"

3. **添加动作** 双击触发器的"动作"区域，在弹出的动作列表中找到 **"特效 - 创建特效到位** **"特效 - 创建指向单位的特**[^1]

4. **配置特效参数** 在弹出的对话框中
   - 选择你想要使用的特效模型文件（通常mdxmdl格式），点击"浏览"按钮选择
   - 设置特效出现的位置（可以是指定坐标点，也可以是某个单位的位置
   - 设定特效持续时间

5. **销毁特* 重要！如果特效不需要永久存在，记得添加 **"特效 - 销毁特** 动作，否则特效会一直保留占用资源[^1]

> **💡 新手提示**：在"创建特效"动作中，选择"指向单位"指定坐标"更常用，因为你可以让特效跟随单位移动，比如让光环一直套在英雄身上

> **⚠️ 常见错误**：新手经常忘记添销毁特动作，导致大量特效堆积在地图上，最终导致游戏卡顿甚至崩溃。养成习惯：创建特效后立即想着它什么时候需要消失

### 音效与音乐播

音效和背景音乐是营造游戏氛围的关键。你可以通过触发器让特定事件播放对应的声音。[^1]

#### 操作步骤

1. **导入音效文件** 在资源管理器中右键点音效"文件夹，选择"导入新资，将你的.mp3wav音效文件添加到地图中

2. **在触发器中创建音效变* 添加动作 **"声音 - 设置变量为声**，浏览选择你导入的音效文件[^1]

3. **添加播放动作** 在触发器动作列表中找**"声音 - 播放声音"**，将刚才设置的音效变量拖入[^1]

4. **等待音效播放完毕（可选）** 如果需要等待音效结束再执行后续动作，使**"声音 - 等待声音"** 动作[^1]

#### 背景音乐播放

对于持续播放的背景音乐，你需要：

1. 创建音效变量并选择音乐文件
2. 添加动作 **"声音 - 播放背景音乐"**，设置是否循环播

> **💡 新手提示**：背景音乐建议使用循环播放，这样玩家在地图中移动时音乐不会中断。普通音效通常只播放一次

> **⚠️ 常见错误**：同时播放太多音效会导致声音混杂。建议使**"声音 - 静音所有声** 先暂停当前音乐，播放剧情对话音效后再恢复

### 摄像机控

摄像机控制可以改变玩家的视角位置和角度，让你的地图有电影般的镜头感

#### 操作步骤

1. **添加摄像机动* 在触发器动作列表中展开"摄像类别[^1]

2. **锁定摄像机到单位** 使用 **"摄像- 锁定摄像机到单位"**，选择一个单位作为摄像机跟随目标[^1]

3. **设置摄像机位* 使用 **"摄像- 设置摄像机位置到区域"**，可以创建过场动画般的镜头移动效

4. **调整摄像机属* 使用 **"摄像- 应用摄像机到玩家"**，可以设置摄像机旋转角度、距离、高度等参数

> **💡 新手提示**：在RPG地图中，建议在单位出生时就将摄像机锁定到玩家英雄身上，否则玩家的视角可能会停留在地图原点，看不到自己的角色

> **⚠️ 常见错误**：摄像机设置过于极端（比如角度旋80度、距离拉得太远）会让玩家头晕。建议先测试不同的数值，找到最舒适的视角

### 小结

- 在地图中播放音效和背景音
- 将摄像机锁定到玩家单位，让视角跟随角色移

建议立即动手练习：创建一个简单的触发器，让英雄释放技能时播放火焰特效和音效，同时摄像机拉近到英雄身上，感受一下这些效果组合在一起的魅力

## 高级动作与实战技

本节将深入讲*变量与数组操**数学与逻辑运算**这两类高级动作，并提供可直接使用*实战模板**。学完本节后，你将能够创建更复杂的游戏逻辑，实现条件判断、循环操作等高级功能

### 变量与数组操

**变量**就像一个可以存放数据的"盒子"*数组**则是存放多个"盒子"架子"。它们是创建复杂触发器的基石[^1]

#### 操作步骤

1. **创建变量** 在触发器编辑器左侧，找到"变量"按钮（形X="的图标）并点击[^2]。在弹出的变量编辑器中，点击"新建变量"，为变量起一个有意义的名字（`HeroHealth`、`PlayerGold`），选择数据类型（整数、实数、单位、点等），然后点击确定

2. **使用"设置变量"动作** 在触发器动作列表中，找到"设置变量"动作[^1]。这个动作的作用是将某个值存入变量中。例如：设置 `HeroHealth = 100`，就是让变量 `HeroHealth` 的值为100

3. **创建数组变量** 在变量编辑器中新建变量时，勾数组"选项[^1]。数组变量用索引（编号）来访问不同的数据槽。比`ItemSlot[1]` 存放第一个物品，`ItemSlot[2]` 存放第二个物品

4. **访问数组元素** 使用变量动作时，在变量名后加方括号填写索引值。例`Set UnitGroup[3] = Triggering Unit`，就是把触发单位存入数组的第3个位置

> **⚠️ 常见错误**：新手经常忘记数组的索引开始（取决于设置）。如果显数组越界"错误，检查你的索引值是否在有效范围内

### 数学与逻辑运算

当你想让触发器根据不同情况做出不同反应时，就需*数学运算**（加、减、乘、除）和**逻辑运算**（与、或、非）[^3]

#### 操作步骤

1. **找到运算动作** 在动作列表的"一分类下，找到"算术运算"动作。这个动作可以让你对整数或实数进行加、减、乘、除计算

2. **执行数学计算** 假设你有一个变`Gold`，想让玩家获0金币，设置动作为：设`Gold = Gold + 10`[^1]。同样，减法用于消费，乘法用于倍数奖励（如双倍经验活动）

3. **使用条件中的逻辑运算** 在触发器条件"部分，你可以组合多个条件[^3]。使（所有条件都满足）（任一条件满足）来创建复杂的判断逻辑。例如：玩家金币00 *** 玩家等级，才能进入某个副本

4. **比较运算** 条件中常用的比较包括：等于、不等于、大于、小于、大于等于、小于等于。这些用于判断数值是否满足要求

> **💡 新手提示**：逻辑运算初学者容易混。记住："是AND（全都要），"是OR（有一个就行）。例年满18*** 持有身份才能买酒，两者缺一不可

### 实战案例与模

下面提供一*"一次性触发器"模板**，常用于任务完成、成就获得等只执行一次的逻辑[^4]

#### 任务只完成一次模

1. **创建布尔变量** `TaskCompleted`，初始值为 `False`[^4]

2. **创建触发任务完成检**，事件设单位死亡"，条件设死亡单位是指定的BOSS"

3. **在动作中添加条件判断**：如`TaskCompleted` 等于 `False`，则
   - 显示"恭喜完成任务提示
   - 设置 `TaskCompleted = True`[^4]
   - 给予玩家奖励（金币、经验、装备）

> **💡 新手提示**：第3步的判断非常重要！没有这个判断，触发器会在每次BOSS死亡时都执行奖励动作，导致奖励被重复发放[^4]
