# RPG英雄系统：经验、升级和属性设

## RPG英雄系统概述

本节将带你认识什么是RPG英雄系统，以及它为什么是所有RPG地图的核心。完成学习后，你将理解英雄、经验值、等级之间的关系，为后续的详细设置打下基础

### 什么是英雄系统

简单来说，英雄系统就是一让玩家控制的角色变强"的规则。在魔兽争霸3中，当你创建一张RPG地图时，你需要告诉电脑：英雄怎么获得经验值、升一级需要多少经验、升级后角色会增加多少力敏捷/智力等属性[^1]

你可以把英雄系统想象游戏角色的成长手——它决定了玩家从开局到游戏结束，角色会经历怎样的成长过程

### 英雄系统对RPG地图的重要

英雄系统是RPG地图的灵魂，原因有三个：

1. **提供目标* 玩家知道自己会变强，有继续玩下去的动
2. **创造差异化** 不同职业/英雄有不同成长路线，让游戏更有趣
3. **支撑战斗系统** 没有合理的属性成长，战斗会变得要么太简单要么太

> **💡 新手提示**：在设计英雄系统前，先问问自玩家玩这张地小时后，角色应该有多强？"这个问题能帮你确定整体设计方向

### 新手常见设计误区

新手在设计英雄系统时容易犯以下错误：

**误区一：经验值曲线失*
- 前期升级太快，后期刷半天不升
- 正确做法是先规划好满级需要多少时间，再反推每级需要的经验

**误区二：属性成长没意义**
- 升一级只点攻击，100级才00点——玩家感觉不到成
- 正确做法是让属性增长与游戏难度匹配，确保后期战斗依然紧

**误区三：三个属性（力量、敏捷、智力）胡乱加点**
- 每个英雄都应该有"主要属，只加主要属性才合理

> **⚠️ 常见错误**：很多新手直接用默认的英雄经验表002060...）[^1]，不做任何调整。但默认表是给传统对战用的，RPG地图节奏完全不同——你需要根据自己地图的大小和预期游戏时长，重新设计经验曲线

### 小结

通过本节，你应该理解了：英雄系统经验值获属性成两部分组成，它决定了玩家角色的强弱变化。在后续章节中，我们将学习如何在World Editor中具体设置这些数值

## 经验值系统设

本节将教你如何在地图中设置英雄的经验值获取系统，让你的英雄能够通过击杀敌人获得经验并升级。学完本节后，你将拥有一个基础的、可正常运作的英雄经验系统

### 基础经验值触发器设置

在魔兽争中，经验值系统其实大部分已经内置好了，但需要你正确配置才能让英雄获得经验。以下是具体步骤

1. **第一步：打开游戏常数设置** 在地图编辑器顶部菜单栏点场景"菜单，然后选择"游戏常数"选项[^1]。这是整个地图最核心的数值配置面板

2. **第二步：找到经验值相关设* 在游戏常数面板左侧列表中，向下滚动找英雄"分类，然后点击展开，找经验值获相关的设置项[^1]

3. **第三步：确认英雄可以获取经验** 确保"启用英雄经验获取"选项处于勾选状态。如果没有勾选，英雄将无法从任何来源获得经验值。这是新手最容易忽略的设置之一

> **⚠️ 常见错误**：很多新手发现自己英雄无法升级，一查原因才发现是这里没有勾选！请务必在配置前检查这个选项

### 经验值表配置与等级上

经验值表决定了英雄升到每一级需要多少经验。魔兽争使用递进式公式，后期升级需要的经验会越来越多[^1]

1. **第一步：设置经验值倍数** 在游戏常数中找到"英雄经验倍数"设置。数.0代表标准难度，你可以根据需要调整（比如想快速升级可以设.0或更高）

2. **第二步：配置等级上限** 游戏常数"中找最大英雄等设置。对于RPG地图，通常建议设置0-25级之间。等级上限太高会让玩家花费大量时间升级，太低则会让游戏节奏过快[^2]

> **💡 新手提示**：如果你想让特定等级之后获得经验的难度变化，可以在地图脚本中使用触发器来修改每级所需经验。关于这一点，我们会在进阶篇详细讲解

### 团队经验与单独经验的区别

这是非常重要但经常被新手混淆的概念

1. **团队经验（Team XP* 当队伍中任意成员击杀敌人时，全队所有英雄都能获得经验值。这适合合作类RPG地图，能让新手玩家不会被老手甩开太远[^4]

2. **单独经验（Individual XP* 只有实际击杀敌人的那个英雄获得经验值。这适合竞技类地图，玩家之间的等级差距会更明显[^4]

> **💡 新手提示**：在大多数RPG地图中，建议使用团队经验模式，这样能让所有玩家都有良好的游戏体验。即使队伍中有新手，他也能跟着其他玩家一起成长

### 小结

完成以上步骤后，你的英雄应该能够正常通过击杀敌人获得经验值并自动升级了。现在你可以进入地图测试，看看英雄击杀怪物后是否能获得经验并升级。如果发现有问题，首先检游戏常数"中的相关设置是否正确

## 属性系统设

属性系统决定了你的RPG英雄"长什么样"——是皮糙肉厚的战士，还是灵活致命的刺客。在这一节，你将学会在World Editor中设置英雄的基础属性类型、让英雄升级时自动获得属性点、以及设计让玩家自由分配属性点的机制。完成学习后，你将拥有一个可以自由加点培养的完整属性系统

### 基础属性类型与作用

1. **第一步：打开游戏常数设置** 点击菜单栏的"游戏菜单，选择"游戏常数"选项[^1]
2. **第二步：找到英雄属性设置区* 在弹出的游戏常数窗口中，找到"英雄"分类，展开后可以看每级力量增长""每级敏捷增长""每级智力增长"等设置项[^6]
3. **第三步：理解三种属性的作用** 力量（STR）增加生命值和生命恢复速度，敏捷（AGI）提升护甲和攻击速度，智力（INT）提高魔法值和技能伤害[^6]
4. **第四步：根据英雄定位调整数* 例如近战英雄可以给更多力量，远程法师给更多智

> **💡 新手提示**：不同英雄应该有不同侧重点。想象你的英雄是"大力还是"大法，然后把对应属性调高

> **⚠️ 常见错误**：把三个属性都设成相同的增长值。这样会导致所有英雄都差不多，失去职业特色

### 属性点获取与分配机

1. **第一步：理解什么是"属性点"** 属性点就像技能点数，英雄升级后获得，可以用来提升力量、敏捷或智力。玩家自己做选择，这自由加点"
2. **第二步：进入触发器编辑器** 在编辑器左侧点击"触发按钮打开触发器编辑器
3. **第三步：创建升级奖励触发* 新建一个触发器，命名为"升级获得属性点"。设事件"单位—一个单位获得等，这样每次英雄升级都会触发这个事件[^2]
4. **第四步：添加加点动作** 在触发器动作"中添游戏—显示提示信息给玩家"显示"恭喜升级！请分配属性点"
5. **第五步：让玩家可以分配属* 创建一个按钮或对话框选项，让玩家选择把属性点加到力量、敏捷或智力上。这需要更复杂的触发器，初学者可以先用简单的"对话实现[^2]

> **💡 新手提示**：先用简单的提示信息让玩家知道升级了，等基础功能稳定后再做完整的加点界面

> **⚠️ 常见错误**：忘记设置触发器条件"。如果条件设置错误，触发器可能会在不该触发的时候触发，甚至导致游戏崩溃

### 属性与战斗力的平衡关系

1. **第一步：了解什么是"战斗** 战斗力是一个综合概念，取决于生命值、攻击力、护甲、攻击速度等多个因素。属性最终要转化为这些战斗数
2. **第二步：参考游戏常数的经验表格** 在游戏常数中找到"英雄经验"设置，可以查看不同等级敌人提供的经验值[^1]
3. **第三步：计算属性收* 每点力量大约增加19点生命值，每点敏捷增加0.16护甲%攻击速度，每点智力增3点魔法值[^6]
4. **第四步：测试并调* 实际玩一局，观察英雄的攻击力和生存能力是否合适。如果玩家觉得太弱，就提高属性增长值；如果太强，就降低

> **💡 新手提示**：制作一张测试地图，专门用来测试英雄属性。用"作弊指令"快速升级，观察效果

> **⚠️ 常见错误**：只看攻击力不看生存。有时候高攻击但血量太少反而让游戏更难——记得平衡！

### 小结

完成以上步骤后，你应该已经在World Editor中设置好了英雄的三种基础属性（力量、敏捷、智力），并且创建了升级奖励属性点的触

## 升级与技能系

本节将教你为RPG英雄配置升级机制和技能学习系统。完成学习后，你的英雄将能够通过战斗获得经验值升级，并解锁新技能！

### 升级事件触发器配

1. **打开触发器编辑器** 在WE顶部菜单栏点触发按钮（一个闪电符号），这是创建自动化逻辑的核心工具[^2]

2. **创建新触发器** 右键左侧空白区域，选择"新建触发，命名为"英雄升级检[^3]

3. **配置事件** 双击触发器，事件"区域点击"新建"，选择"单位 - 单位获得经验。这表示当任意单位获得经验时触发此逻辑[^4]

4. **设置条件** 添加条件：单- 拥有指定属性是"英雄"，确保只有英雄升级才触发后续动作[^5]

5. **添加动作** 动作"区域添加
   - 变量 - 设置整数 = "经验等级" = (单位等级 of (触发单位))
   - 显示升级提示文本给玩

> **💡 新手提示**：触发器的执行顺序很重要！如果你的升级触发器没有响应，先检查它是否被放在了其他触发器的前面

> **⚠️ 常见错误**：新手经常忘记设条件"导致所有单位（包括小兵）都触发升级事件。务必记得添单位是英的判断条件[^5]

### 技能学习与技能点分配

1. **打开单位编辑* 按F6或菜单栏"编辑 - 单位"进入单位属性配置[^2]

2. **选择英雄单位** 在左侧找到你的英雄模板，点击编辑其属性[^3]

3. **配置技能栏** 科技标签页中，为英雄添加可学习的技能。点击空白格子，选择你创建的技能[^6]

4. **设置技能点** 高级 - 英雄属中，修改"每级技能点"数值（通常设为1）[^4]

5. **创建技能学习触发器** 回到触发器编辑器，新学习技触发器：
   - 事件：单- Unit Learns Skill
   - 动作：设置该单位学会了对应技

> **💡 新手提示**：游戏默认的英雄升级公式会将经验要求逐级递增[^1]，如果觉得升级太慢，可以游戏常数"中调整经验表格数值

### 被动技能与主动技能设

1. **区分两种技能类* 主动技能需要玩家手动释放，被动技能自动生效。选择技能模板时，注可施被动"两个属性的区别[^6]

2. **创建被动技* 新建技能时，将"被动"属性设，这种技能无需手动释放，属性加成会在英雄获得时自动生效[^6]

3. **创建主动技* 保持"可施，然后配置：
   - 施法时间（施放前摇）
   - 冷却时间
   - 魔法消
   - 效果区域范围

4. **设计技能等* 双击技能图标可以设置每级属性变化，高等级技能通常有更强的效果[^2]

> **⚠️ 常见错误**：新手常常忘记给技能设置正确的"数据 - 允许的目施法距离"，导致技能无法正常使用。创建完技能后务必仔细检查各项数值！

### 小结

完成以上配置后，你的RPG地图已经具备了基本的英雄成长系统。玩家可以通过击杀敌人获得经验，升级时自动获得技能点，并从技能栏中选择学习主动或被动技能。建议先用测试模式（按F10运行地图）验证升级和技能是否正常工作，再进行平衡性调整！

## 平衡性调整与测试

在本节中，你将学习如何对自己设计的英雄经验、升级和属性系统进*平衡性调整和测试**。学完本节后，你将能够通过实际游戏测试发现问题，并根据测试结果优化你的英雄系统，让玩家拥有良好的游戏体验

### 前期数值设计与调整

英雄的数值设计是平衡性的基础。在动手之前，你需要先规划好核心数值：

1. **打开触发编辑* 在WE菜单栏点触发（触发器是控制游戏逻辑的代码集合，新手先用默认设置即可）[^2]

2. **设置经验值常* 进入"游戏 - 游戏常量"面板，找英雄经验获取"相关选项[^1]

3. **调整经验表格** 在经验表格中填入每个等级需要的经验值。例如：1级英雄击杀获得100点经验，260点，320点，以此类推[^1]

4. **规划属性成* 决定每个英雄每升一级获得多少力量、敏捷、智力属性加成[^6]

> **💡 新手提示**：建议先参考魔兽争原版英雄的经验曲线设计，再根据自己的RPG类型调整。原版经验表是一个很好的起点[^1]

> **⚠️ 常见错误**：很多新手设置的初期经验需求太低，导致玩家升级太快，中后期内容还没体验到就满级了。建议前期每级经验增长幅度控制在30%-50%左右

### 游戏测试与迭代优

设计完数值后，必须通过实际游戏来验证效果：

1. **保存并进入测试模* 按F6或在WE中点测试地图"按钮，启动游戏[^2]

2. **使用测试指令** 在游戏中输入"-lvlup"指令给英雄升级，观察属性变化是否符合预

3. **记录测试数据** 建议制作一个简单的表格，记录：当前等级、获得的经验、升级所需时间、角色属性数

4. **重复测试循环** 根据发现的问题返回WE调整数值，再次进入测试模式验证[^4]

> **💡 新手提示**：至少测-5次完整的升级流程，记录每次的变化趋势，这样才能发现数值设计中的规律性问题

### 玩家反馈与系统改

自己测试只能发现部分问题，真正的平衡需要玩家参与：

1. **邀请测试玩* -3位朋友试玩你的地图，记录他们反馈的问

2. **关注核心问题** 重点收集：升级速度是否太慢/太快、属性成长是否让某些职业过强、玩家是否感到无聊或挫败

3. **分批次优* 不要一次性改动太多数值，每次只调-2项，便于判断改动效果[^3]

4. **持续迭代** 地图平衡是一个长期过程，保持开放心态接受批评和建议

> **⚠️ 常见错误**：新手容易在收到负面反馈后完全推翻自己的设计。其实玩家吐槽是好事，说明他们在认真玩你的地图！先记下问题，再判断是否需要修改

### 小结

完成以上三个步骤后，你应该：
- 建立了合理的英雄经验曲线和属性成长表
- 完成了至轮自我测试并记录了数
- 收集了真实玩家的反馈意见
- 根据反馈进行了有针对性的数值调

平衡性调整是一*持续的过*，不要指望一次设计就完美。随着测试和反馈的积累，你的英雄系统会越来越完善！

4. **第四步：复制需要的部分** 选中你喜欢的功能触发器，右键选择"复制"，然后粘贴到自己的地图中使用

> **💡 新手提示**：下载地图后建议另存一份副本再修改，避免损坏原始文件。另外，部分国外网站需要代理才能访问，可以尝试使用浏览器翻译功能

> **⚠️ 常见错误**：很多新手直接用别人的完整地图改皮肤，却不知道触发器可能很复杂。务必耐心阅读触发器注释（黄色文字说明），理解每一步的作用后再修改

### 从零到一的完整案例解

让我们用一个简单案例来完整走一遍英雄系统的搭建流程。假设我们要制作一个能通过杀怪获得经验、升级后提升攻击力的战士英雄

1. **第一步：创建英雄单位** 在对象编辑器中新建一英雄"类型的单位，设置其主属性为"力量"，初始等级为1[^6]
2. **第二步：配置经验值表** 在游戏性常量中找到"英雄经验，设-5级的经验需求为10000505000点[^1]
3. **第三步：设置属性成* 在英雄单位的"升级后获属性中，添加每级力2、敏1、智1的成长
4. **第四步：测试验证** 按F9运行地图，用英雄攻击野怪，观察经验条变化和属性提升是否符合预
5. **第五步：记录与优* 把测试结果记录下来，如果发现升级太快或太慢，回到游戏性常量调整数值重新测

> **⚠️ 常见错误**：新手容易忽游戏性常中的"经验因子"设置，导致英雄升级速度与预期不符。务必检查这个参数是否为1（默认值）[^1]
