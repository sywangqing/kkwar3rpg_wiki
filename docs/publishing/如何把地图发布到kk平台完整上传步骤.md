# 如何把地图发布到KK平台：完整上传步骤

## 概述

在正式开始上传地图之前，我们需要先了解整个发布流程是什么样的。本节将为你介绍 KK 平台地图发布的基本概念，包括什么是"审核版本"、发布前需要准备什么，以及为什么地图不能立刻上线。这些知识看起来有点"虚"，但**理解这些概念可以帮你避免后面很多坑**[^1]！

当你学完本节后，你会对 KK 平台的地图发布流程有一个整体认识，知道"发布"和"提交审核"有什么区别，为后续的具体操作做好准备。

### 为什么要先了解这些？

当你把地图上传到 KK 平台后，地图并不会立刻出现在玩家面前供所有人玩到。**平台需要先审核你的地图**，确保内容符合规范[^1]。这个审核过程可能需要一段时间，期间你可以修改和重新提交。

> **💡 新手提示**：把审核想象成"老师批改作业"。你交上去后，老师要检查有没有问题，通过了才会给你小红花（发布资格）。

### 发布地图的完整流程概览

虽然具体操作在后面几节会详细讲，但先来认识一下整体流程：

1. **上传地图文件** — 把你的 .w3x 地图文件上传到 KK 开发者平台
2. **提交审核版本** — 平台会对你的地图进行审核
3. **等待审核通过** — 审核通过后，你可以选择发布到线上
4. **地图上线** — 玩家就能在 KK 平台找到并玩你的地图了

> **⚠️ 常见错误**：很多新手以为"上传就等于发布"，以为传上去玩家就能玩了。实际上，**上传只是第一步，还需要等待审核通过后才能正式发布**[^1]。如果审核不通过，你需要修改后重新提交。

### 什么是"审核版本"？

KK 平台有一个重要概念叫"审核版本"。简单理解就是：**同一时间，你的地图只能有一个版本处于审核状态**[^1]。

举个例子：
- 假设你提交了 v1.0 版本审核中
- 然后你又修改了地图，想提交 v1.1
- 这时 v1.0 会**被 v1.1 覆盖**，重新开始审核

> **💡 新手提示**：这就是为什么建议你在提交审核前，先在本地测试充分。因为一旦提交新版本，旧的审核就会重新开始，会耽误时间。

### 小结

现在你已经了解：

- 地图发布不是"立刻上线"，需要经过审核流程
- 上传后只有"审核通过"的版本才能选择发布
- 同一时间只能有一个审核版本在审核中
- 建议先在本地测试充分后再提交审核
