# 我的第一个触发器：让单位说话

## 什么是触发器？——单位的“嘴巴”

想象一下，你创造了一个帅气的英雄走进村庄，但无论你点击他多少次，他都一言不发，像个哑巴一样站在原地。这听起来是不是很无聊？**触发器（Trigger）**就是给这个“哑巴”装上嘴巴的东西——它能让单位根据玩家的操作做出反应，说出你想让它们说的话！在本节，你将理解触发器到底是什么，以及它为什么能让你的地图从“一潭死水”变成“活”的世界。

### 为什么触发器让地图更有趣

在地图编辑器里，如果你不给单位添加任何触发器，那么一张地图建好后就永远保持原样——单位不会动，对话不会发生，敌人不会主动攻击你。这种地图玩起来就像一张会动的图片，非常枯燥。

触发器就像给地图添加了**“智能”**[^2]。当你做了某个动作（比如点击一个单位、进入某个区域、时间到达某一刻），触发器就会“注意到”这件事，然后自动执行你预设好的反应——比如让单位开口说话，或者释放一个技能。没有触发器，你的地图就是一条笔直的路；有了触发器，你的地图才是一条充满惊喜的探险之旅。

### 触发器能做哪些事？

每个触发器都像一个小小的“**如果……那么……**”的指令。一个完整的触发器包含三个部分，简称**ECA结构**[^2]：

- **E = Event（事件）**：什么时候触发？比如“当玩家点击单位时”
- **C = Condition（条件）**：要不要触发？比如“只有当单位是英雄时才说话”
- **A = Action（动作）**：触发后做什么？比如“显示对话气泡说：'你好，冒险者！'”

### 操作步骤

让我们先认识一下触发器编辑器长什么样：

1. **打开触发器编辑器** — 在WE（World Editor，世界编辑器）界面底部菜单栏找到📌图标按钮，点击它就能打开触发器编辑器窗口[^3]

> **💡 新手提示**：触发器编辑器默认可能没有打开，记得先找到这个按钮！不同版本的编辑器这个图标位置可能略有不同，如果找不到，可以在菜单栏的“模块”选项里手动勾选“触发器编辑器”。

> **⚠️ 常见错误**：很多新手以为触发器是某种“代码”或“编程”，其实完全不是！触发器就是一系列**设置选项**，你只需要点鼠标选择各种参数，不需要写任何代码。

### 小结

现在你应该已经明白：**触发器就是让单位“活”起来的魔法指令**。它由事件（什么时候）、条件（要不要）、动作（做什么）三部分组成[^2]。当你创建了触发器，你的地图就不再是静态的画面，而是一个会对玩家做出反应的有趣世界！下一节，我们将实际动手，创建你的第一个触发器——让单位说出第一句话！

## 打开触发器编辑器

本节我们将学习如何打开触发器编辑器，并认识它的基本界面。完成本节后，你将能够找到并打开触发器编辑器，看到它由哪些部分组成，为后续学习触发器编写打下基础。

### 操作步骤

1. **启动世界编辑器** — 在桌面或开始菜单中找到"World Editor"（世界编辑器）并双击打开。如果你使用的是中文版本，可能显示为"魔兽争霸3地图编辑器"[^1]。

2. **定位触发器编辑器入口** — 在世界编辑器顶部菜单栏中，找到"模块"（Modules）菜单，然后将鼠标悬停在上面，会展开一个下拉列表[^2]。

3. **点击"触发器编辑器"** — 在展开的模块列表中，找到并点击"触发器编辑器"（Trigger Editor）选项[^2]。这是编辑器的核心模块之一，专门用于创建和管理各种游戏事件与响应。

4. **等待编辑器加载** — 触发器编辑器会作为一个独立面板出现在世界编辑器窗口中。如果这是你第一次打开，可能会显示一个空的触发器列表。

> **💡 新手提示**：触发器编辑器可以与地形编辑器等其他模块同时打开。你可以在世界编辑器的主界面同时进行多项编辑工作，窗口布局会自动调整适应。

> **⚠️ 常见错误**：有些新手在"视图"（View）菜单中寻找触发器编辑器，而不是"模块"菜单。记住：**触发器编辑器是一个独立的模块**，需要在"模块"菜单中打开，而不是"视图"菜单。

### 触发器编辑器的界面介绍

当你成功打开触发器编辑器后，会看到以下主要区域：

**工具栏区域**（顶部）
从左到右依次是：变量编辑器、新建分类、新建触发器、新建注释触发器、新建事件、新建条件、新建动作按钮[^3]。这些是编写触发器时最常用的工具。

**触发器列表面板**（左侧）
这是最核心的区域，显示地图中所有的触发器列表。你可以在这里看到现有的触发器，选中它们进行编辑或查看。

**触发器编辑区域**（右侧或中央）
当你选中一个触发器后，这里会显示该触发器的详细内容，包括事件（Event）、条件（Condition）和动作（Action）三个组成部分[^2]。

### 小结

完成以上步骤后，你应该能够：
- ✅ 在世界编辑器中找到并打开触发器编辑器
- ✅ 看到触发器编辑器的工具栏和列表面板
- ✅ 了解触发器编辑器的基本布局

下一节我们将学习如何创建一个全新的触发器，让单位开口说话！

## 触发器的三个组成部分

在这一节里，你将学习触发器的**核心结构**——它由三个必不可少的部分组成。理解这个结构后，你就能读懂几乎所有触发器的逻辑，也能开始自己编写简单的触发器了。

### 事件（Event）——触发器什么时候启动

**事件**就像是触发器的"开关"，告诉地图"什么时候该开始做点什么"。

> **💡 新手提示**：可以把事件想象成"闹钟"——闹钟响了，你才会起床（执行动作）。没有设定闹钟，你就永远不会主动起床。

事件有很多种，比如：
- 单位进入区域时
- 单位死亡时
- 玩家聊天输入特定文字时
- 游戏时间到达某一刻时

### 条件（Condition）——什么时候才真正执行

**条件**是"过滤器"，只有条件满足时，触发器才会真正执行动作。

> **⚠️ 常见错误**：新手经常混淆"事件"和"条件"。记住：事件是"什么时候开始"，条件是"符不符合要求"。即使事件触发了，如果不满足条件，动作也不会执行。

举个例子：事件是"单位死亡"，条件可以是"死亡的单位是英雄"——这样只有英雄死亡时才会触发，不是所有小兵死亡都触发。

### 动作（Action）——触发器具体做什么

**动作**就是触发器要执行的具体操作，比如：
- 让单位说话（显示对话框）
- 创建或删除单位
- 修改单位属性
- 播放音效或特效

### 小结

触发器的结构就像一个**因果链条**：

| 部分 | 作用 | 类比 |
|------|------|------|
| 事件 | 什么时候开始 | 闹钟响了 |
| 条件 | 符不符合要求 | 是不是该起床了 |
| 动作 | 具体做什么 | 起床穿衣 |

记住**ECA**这三个字母，这就是所有触发器的基础结构[^2]。只要掌握了这个框架，你看任何触发器都能找到它的"开关"、"过滤器"和"操作"三部分。

## 创建“单位说话”触发器

本节我们将学习创建一个让游戏中的单位自动说话的触发器。学完之后，你就能让任意单位在游戏里显示气泡对话框，比如NPC开场介绍、任务提示等常见功能。

### 操作步骤

1. **打开触发器编辑器** — 在WE主界面底部工具栏找到并点击“触发器”按钮（一个闪电图标形状的按钮）[^3]。这会打开触发器编辑器窗口，里面有左侧的触发器列表和右侧的编辑区域。

2. **创建新触发器** — 在触发器编辑器中，点击左上角的“新建触发器”按钮（绿色加号图标）[^3]。系统会自动生成一个空白触发器，默认名字叫“Trigger”。你可以右键重命名为“单位说话演示”方便辨认。

3. **添加“单位说话”动作** — 展开你刚创建的触发器，右键点击“事件”下方的空白区域，选择“新建事件”。然后在弹出的动作列表中找到“通讯”分类，展开后双击“单位说话”选项[^1]。这个动作就是让单位显示气泡对话框的核心功能。

4. **设置说话单位** — 添加动作后，你会在右侧看到三个参数框。第一个参数框“单位”需要指定谁来说话。点击参数框，从下拉菜单中选择“触发单位”（表示使用这个触发器触发的那个单位）。你也可以点击“创建单位”临时创建一个测试用的单位。

5. **编写说话内容** — 第二个参数框“消息内容”填写单位要说的话，直接输入中文文本即可，比如“欢迎来到我的商店！”。第三个参数框“显示时长”建议保持默认的3秒（表示对话框显示3秒后自动消失）[^2]。

> **💡 新手提示**：如果你想让单位持续说话，可以在触发器中添加“等待”动作（比如等待5秒），然后再添加一个“单位说话”动作，这样单位会轮流说出多句话。

> **⚠️ 常见错误**：新手经常忘记设置触发条件，直接运行地图后单位没有反应。这是因为“单位说话”动作需要配合“事件”才能执行——你需要先添加一个事件，比如“地图初始化”或“单位进入区域”，这样触发器才知道什么时候让单位开口说话[^2]。

### 小结

完成以上步骤后，你的第一个触发器就创建成功了。运行地图测试一下，如果设置正确，你指定的单位会在相应事件触发时显示气泡对话框并说出你编写的内容。现在你可以尝试修改文字内容，或者添加多个“单位说话”动作让单位说一整段台词了！

## 保存并测试你的触发器

你已经成功创建了第一个触发器！本节将教你如何保存地图并在游戏中测试你的触发器效果。学完之后，你就能亲眼看到你写的触发器在游戏中运行了。

### 保存地图的正确方法

1. **第一步：保存地图** — 按下键盘快捷键 `Ctrl + S`，或者点击顶部菜单栏的"文件(File)" → "保存(Save)"[^1]。这是最保险的保存方式。

2. **第二步：给地图起个名字** — 如果是首次保存，编辑器会弹出对话框让你输入地图名称。建议起一个简单好记的名字，比如"我的第一个地图"。

3. **第三步：确认保存成功** — 保存完成后，地图文件会出现在你指定的文件夹中，文件后缀是`.w3x`（重置版）或`.w3m`（经典版）。

> **💡 新手提示**：养成随时按`Ctrl+S`保存的习惯！编辑过程中很容易不小心关闭编辑器，之前的工作就白费了。

### 在游戏中测试触发器

1. **第一步：点击测试按钮** — 在编辑器工具栏找到"测试地图(Test Map)"按钮，看起来像一个播放箭头 ▶，或者直接按键盘 `F10`[^3]。

2. **第二步：等待游戏加载** — 等待几秒钟，编辑器会自动启动魔兽争霸3游戏并加载你的地图。

3. **第三步：进入游戏测试** — 进入游戏后，按照你设置的触发条件进行操作。例如，如果你的触发器是"单位说话"，就选中你放置的单位，看看是否有文字气泡出现。

4. **第四步：退出测试** — 测试完成后，按 `Esc` 键或输入 `-退出` 命令返回编辑器，继续修改。

> **⚠️ 常见错误**：很多新手发现触发器没反应就直接关游戏重开，其实应该先看触发器面板左下角的"最后运行的触发器"状态，确认触发器是否真的被激活了[^2]。

### 触发器不生效？常见原因排查

如果你的触发器在游戏中没有任何反应，按以下顺序检查：

| 检查项 | 操作方法 |
|--------|----------|
| 触发器是否启用 | 看触发器名称前面有没有" ✓ "勾选，没有的话点一下启用 |
| 事件是否正确 | 确认你设置的事件条件在游戏中确实会发生 |
| 条件是否满足 | 检查CONDITION（条件）里设置的条件是否都达成 |
| 动作是否添加 | 确认ACTION（动作）里至少有一个动作，没有动作的触发器什么都不会做 |

> **💡 新手提示**：选中出问题的触发器后，按住`Shift`键的同时按`Enter`，可以打开触发器的运行日志，查看它每次运行时具体执行了哪些步骤，非常适合排错。

### 小结

完成以上步骤后，你应该已经成功保存了你的地图，并在游戏中看到了触发器的效果。如果触发生效了——恭喜你完成了第一个触发器！如果还没有生效，对照上表的排查清单逐一检查，很快就能找到问题所在。触发器调试是每个地图作者的必备技能，多练习几次你就会越来越熟练。

## 让单位说话更有趣

在前面的课程中，你已经学会了让单位说话的基本方法。现在我们来学习两个让对话更加生动有趣的技巧：添加音效和根据不同情况说不同的话。

### 添加音效让说话更生动

纯文字对话有时候显得有点单调。让我们给它加点"声音"！

1. **第一步：找到"显示警告消息"动作** — 在触发器编辑器中，找到你之前创建的触发器，双击"显示警告消息"动作将其打开[^1]
2. **第二步：添加音效参数** — 点击动作右侧的三角符号，在下拉菜单中找到"播放音效"选项并勾选[^3]
3. **第三步：选择音效文件** — 点击"无音效"文字，选择一个合适的.wav或.mp3音效文件。你可以搜索"attack"、"death"等关键词快速找到战斗相关音效
4. **第四步：测试** — 按F6测试地图，当触发执行时，你应该能听到音效和文字同时出现

> **💡 新手提示**：音效文件不宜过大，建议控制在500KB以内，否则会影响游戏加载速度。你可以用Audacity等免费软件压缩音频。

### 让单位根据不同情况说不同的话

有时候我们希望同一个触发器，根据不同情况显示不同的内容。这就需要用到"如果/那么/否则"逻辑。

1. **第一步：添加条件判断动作** — 在触发器中点击"新建动作"，找到"如果/那么/否则"动作[^2]
2. **第二步：设置判断条件** — 点击"如果"后面的"<无法比较>"，设置你的条件。比如比较"触发单位生命值 < 50"表示单位血量低于50%
3. **第三步：编写"那么"分支** — 在"那么"下面添加"显示警告消息"，输入低血量时要说的话，比如"我快撑不住了！"
4. **第四步：编写"否则"分支** — 在"否则"下面添加另一个"显示警告消息"，输入正常情况下要说的话，比如"我状态良好！"
5. **第五步：测试** — 让你的单位满血和低血时分别触发，看看是否显示不同内容

> **⚠️ 常见错误**：新手经常忘记在"否则"分支里添加动作，导致不满足条件时什么都不发生。检查一下每个分支下面都有内容！

### 小结

恭喜你！现在你已经掌握了让单位说话的两个进阶技巧。添加音效可以让对话更生动，而条件判断则让你的触发器能够"思考"——根据不同情况做出不同反应。建议先从简单的音效开始练习，再尝试添加条件判断。
