# 制作对话框和选项菜单

## 对话框基础概念

本节将带你认识**对话框**这个工具。学完本节后，你将理解对话框是什么，以及它能帮你实现哪些游戏功能。

### 什么是对话框

**对话框**是魔兽争霸编辑器中一种**自定义的弹出窗口**，你可以把它理解成一张"小卡片"，上面可以放文字、图片和按钮。玩家点击按钮后，会触发相应的事件（这个概念后面触发器章节会详细讲）[^1]。

简单来说，对话框就是**你和玩家"对话"的窗口**——你想让玩家看到什么信息、让玩家做什么选择，都可以通过对话框来实现。

### 对话框与游戏默认菜单的区别

游戏默认菜单（如回车后输入 -ap 这种）是由游戏本身预先设定好的，**你无法修改它们的内容和样式**。而自定义对话框是你自己创建的，完全由你控制：

| 对比项 | 游戏默认菜单 | 自定义对话框 |
|--------|-------------|-------------|
| 能否修改内容 | ❌ 不能 | ✅ 可以自由设置 |
| 能否添加按钮 | ❌ 不能 | ✅ 可以添加多个按钮 |
| 触发时机 | 系统控制 | **由你通过触发器控制** |

打个比方：游戏默认菜单像是别人印好的表格，你只能填选项；而自定义对话框是你自己画的纸，想写什么写什么。

### 对话框的常见使用场景

在 RPG 地图中，对话框最常用的场景包括：

1. **选择英雄** — 玩家进入游戏后，弹出对话框让玩家选择想用的英雄[^2]
2. **显示任务提示** — 任务开始时弹出提示框说明任务目标
3. **商店购买界面** — 玩家与NPC对话时弹出选项菜单
4. **游戏设置** — 让玩家选择难度、模式等

> **💡 新手提示**：对话框本身只是"显示"给玩家看的内容，它不会自动产生任何效果。你需要配合**触发器**（一种"如果...那么..."的逻辑工具）来告诉编辑器：玩家点击某个按钮后应该发生什么。

> **⚠️ 常见错误**：很多新手以为创建了对话框就会自动显示，但实际需要用触发器来"Show"（显示）这个对话框，否则玩家根本看不到它。对话框和触发器的关系就像"画"和"画笔"——光有画没有笔，画不会出现在墙上。

### 小结

本节你需要记住三个要点：
- 对话框是**自定义的弹出窗口**，可以放文字和按钮
- 它与游戏默认菜单的区别在于**你可以完全控制**
- 对话框需要配合**触发器**使用，否则不会自动显示

下一节我们将学习如何实际创建一个对话框！

## 创建对话框

对话框是玩家与游戏互动的"窗口"——想想看，当你想让玩家选择英雄、确认操作或显示任务提示时，就需要用到对话框。在本节，你将学习如何在触发器编辑器中创建第一个对话框，并为其设置标题。学完后，你就能让游戏主动弹出信息窗口与玩家交流了。

### 操作步骤

1. **打开触发器编辑器** — 在地图编辑器主界面，找到顶部菜单栏的"触发器"按钮并点击（或直接按F4键）[^4]。编辑器会打开一个新窗口，这就是我们编写游戏逻辑的地方。

2. **创建新触发器** — 在触发器编辑器左侧，找到"触发器"文件夹，右键点击 → "新建触发器"[^5]。建议把它的名字改成"对话框示例"，方便以后管理。触发器就像一个"任务清单"，里面的一条条指令会按顺序执行。

3. **添加"创建对话框"动作** — 选中你新建的触发器，在右侧窗口点击"新建动作"按钮。在动作列表中找到"对话框"分类，展开后选择"创建对话框"[^1]。点击确定后，这个动作就会被添加到触发器中。

4. **保存对话框到变量** — 聪明的做法是：创建对话框后，立刻把它保存到一个"变量"中（变量就像一个带名字的储物箱，可以随时取出里面的东西）。在"创建对话框"动作上右键，选择"设置变量"，给变量起名比如"HeroDialog"[^2]。这样后面才能再次使用这个对话框。

### 设置对话框标题和样式

5. **添加"修改对话框标题"动作** — 再次点击"新建动作"，在对话框分类中找到"修改对话框标题"[^2]。在弹出的设置窗口中，选择你刚才保存的变量作为目标对话框，然后在"新标题"框中输入你想显示的文字，比如"请选择你的英雄"。

6. **添加"显示对话框"动作** — 对话框创建后默认是隐藏的，必须手动显示！在动作列表中选择"显示对话框"，设置对话框和要显示给哪位玩家（通常是"触发玩家"）[^3]。

> **💡 新手提示**：变量名的命名建议用英文+驼峰格式，比如`HeroDialog`、`MainMenuDialog`，这样代码看起来更专业，也方便以后阅读。

> **⚠️ 常见错误**：很多新手创建了对话框但游戏里看不到——八成是忘了添加"显示对话框"动作！还有同学把对话框变量写成局部变量，结果其他触发器用不了。解决办法：把对话框变量设置为"全局变量"（在编辑器顶部变量列表中创建）。

### 小结

完成以上步骤后，你的触发器应该包含三个动作：创建对话框 → 保存到变量 → 显示对话框。按F6测试地图时，按下你设置的快捷键（或者根据你后续添加的事件），对话框就会出现在屏幕上！不过目前对话框是空的，下一节我们将学习如何向对话框添加按钮，让玩家能够真正做出选择。

## 向对话框添加元素

本节将学习如何向对话框添加按钮和文本内容。学完本节后，你将能够创建一个包含选项按钮的完整对话框，让玩家可以进行选择。

### 添加按钮

1. **打开触发器编辑器** — 在世界编辑器主界面，点击上方的"触发器编辑器"按钮（像一个闪电符号）[^4]
2. **选择对话框触发器** — 在左侧触发器列表中，找到你之前创建的触发器（比如"初始化对话框"），双击打开它
3. **添加按钮动作** — 在触发器动作列表下方，右键选择"新建动作"，然后在搜索框输入"创建按钮"[^1]
4. **设置按钮尺寸和位置** — 找到"对话框 - 创建用于对话框的按钮"动作，设置以下参数[^1]：
   - **尺寸**：宽度200，高度50（像素）
   - **锚点**：选择"左上"（表示从左上角开始计算位置）
   - **偏移**：X填50，Y填500（按钮距离对话框左边缘50像素，上边缘500像素）
5. **保存到变量** — 将新创建的按钮保存到一个变量中，方便后面检测玩家点击了哪个按钮

> **💡 新手提示**：每个按钮都应该保存到独立的变量中。比如你有3个选项，就创建3个"对话框按钮"类型的变量，分别命名为"按钮选项1"、"按钮选项2"、"按钮选项3"。

### 添加文本内容

1. **添加改变标题动作** — 还是在同一个触发器中，右键新建动作，搜索"改变标题"[^2]
2. **设置对话框标题** — 在动作参数中选择你之前创建的对话框变量，标题文本框中输入你想显示的文字，比如"请选择一个英雄"[^2]
3. **设置按钮显示文字** — 在创建按钮的动作中，找到"按钮文本"参数，输入按钮上要显示的文字，比如"火焰法师"或"Pick Hero"[^1]

> **⚠️ 常见错误**：新手经常忘记给按钮设置文字，导致按钮上显示为空。一定要记得在创建按钮时填写"按钮文本"参数，否则玩家看到的就是空白按钮，不知道每个选项是什么。

### 小结

完成以上步骤后，你的对话框已经具备了基本的元素：
- 一个显示标题，告诉玩家要做什么
- 至少一个按钮，供玩家点击选择

你现在可以测试地图，看看对话框是否正常显示。如果要添加更多选项，只需重复"添加按钮"的步骤，为每个按钮创建独立的变量即可。后续我们还会学习如何检测玩家点击了哪个按钮，并执行相应的操作。

## 显示和隐藏对话框

本节将学习如何把已经创建好的对话框显示给玩家看，以及如何在不需要时隐藏或彻底销毁对话框。学完本节后，你的游戏就能在关键时刻弹出选项菜单让玩家做出选择了！

### 操作步骤

#### 如何向玩家显示对话框

1. **打开触发器编辑器** — 在WE主界面按 **F4** 键，或点击顶部菜单「触发器」打开触发编辑器窗口[^1]

2. **选择或创建触发器** — 在左侧触发器列表中，选中你要用来显示对话框的触发器（如果没有就新建一个）

3. **添加「显示对话框」动作** — 在触发器的「动作」区域点击右键，选择「对话框」→「显示对话框」[^2]

4. **设置参数** — 点击刚添加的动作，在弹出的属性框中：
   - 选择你要显示的 **对话框**（从下拉菜单选取你之前创建的对话框变量）
   - 选择要显示给哪个 **玩家**（通常选「触发玩家」或「所有玩家」）

> **💡 新手提示**：记得检查你的触发条件！显示对话框的触发器必须有对应的事件（比如玩家点击某个单位、按ESC键、或游戏时间到达等），否则对话框永远不会出现[^3]

#### 如何隐藏对话框

5. **添加「隐藏对话框」动作** — 在同一触发器（或新的关闭触发器）中，添加动作「对话框」→「隐藏对话框」[^2]

6. **设置隐藏参数** — 选择要隐藏的是哪个对话框，以及对哪个玩家隐藏

> **⚠️ 常见错误**：很多新手只隐藏对话框但不销毁它！隐藏后对话框仍然存在于内存中，如果玩家再次打开相同对话框，可能会出现重叠或显示错误。**建议在隐藏后等待一小段时间（如0.5秒），再用「清除对话框」动作彻底销毁它**[^4]

#### 如何彻底销毁对话框

7. **添加「清除对话框」动作** — 在触发器中添加「对话框」→「清除对话框」动作[^2]

8. **设置清除目标** — 选择要清除的对话框变量

### 小结

完成以上步骤后，你应该能够：

- ✅ 在游戏过程中向指定玩家显示对话框
- ✅ 隐藏已显示的对话框
- ✅ 彻底销毁不需要的对话框释放内存

你现在可以让玩家在游戏中看到你精心设计的选项菜单了！记得测试时用 **「测试地图」** 功能（Ctrl+F9）来查看效果。

## 响应玩家选择

本节将教你如何让游戏"听懂"玩家的选择。当玩家点击对话框中的按钮时，你需要告诉游戏：根据不同的按钮，执行不同的操作。学完本节后，你将能够制作一个完整的"选择英雄"功能。

### 操作步骤

1. **打开触发器编辑器** — 在地图编辑器顶部菜单中点击"触发器"（Trigger）图标，打开触发器编辑窗口[^4]

2. **新建触发器** — 在左侧触发器列表中右键点击，选择"新建触发器"，给它起个名字，比如"响应对话框点击"

3. **设置触发事件** — 在触发器编辑器中，点击"事件"（Event）区域，展开"对话框"（Dialog）类别，选择"对话框 - 被点击"事件[^2]。这一步的意思是："当任何对话框的按钮被点击时，执行这个触发器"

4. **添加条件检查** — 在触发器中点击"条件"（Condition）区域，添加一个新条件。选择"对话框比较"，然后设置条件为：
   - 第一个值：点击的对话框按钮
   - 比较符：等于
   - 第二个值：你之前创建的按钮变量[^2]
   
   为什么要这样写？因为一个触发器可能被多个按钮共用，你必须告诉游戏："只有当这个特定的按钮被点击时，才执行后面的动作"

5. **添加执行动作** — 在触发器的"动作"（Action）区域，添加你想执行的操作。比如：
   - 显示另一对话框
   - 创建某个单位
   - 隐藏当前对话框
   - 显示消息告诉玩家"你选择了XXX"[^2]

6. **为其他按钮重复** — 如果你有多个选项按钮，可以复制这个触发器，然后修改条件中的按钮变量，为每个按钮创建独立的响应逻辑

> **💡 新手提示**：在设置条件时，一定要选择正确的按钮变量。如果你在创建按钮时没有保存到变量，可以返回上一节查看如何将按钮存储到变量中。

> **⚠️ 常见错误**：新手经常忘记添加条件检查，导致点击任意按钮都会触发所有动作。另一个常见错误是：条件中选择了错误的按钮变量，比如设置了"按钮A的条件"却触发了"按钮B的动作"。

### 小结

完成以上步骤后，当你运行地图测试时，点击对话框中的按钮，游戏就会根据你设置的逻辑做出相应反应。比如点击"选择战士"按钮，系统会创建战士角色并显示"你选择了战士"；点击"选择法师"按钮，则会创建法师角色。这样，一个简单的"选择英雄"功能就完成了！

## 实用示例与进阶技巧

本节将通过三个实用案例，手把手教你制作游戏设置菜单、确认对话框和多选项菜单。完成学习后，你将能够为地图添加专业的交互界面，让玩家通过按钮完成各种选择操作。

### 制作游戏设置选项菜单

游戏设置菜单可以让玩家在游戏开始前调整音量、游戏速度等选项。

1. **第一步：创建对话框** — 在触发器编辑器中，点击"对话框"分类，选择"创建对话框"动作[^1]。这会创建一个空白的对话框容器。

2. **第二步：添加标题** — 使用"更改对话框标题"动作，设置标题为"游戏设置"。标题会显示在对话框顶部，告诉玩家这个菜单的作用。

3. **第三步：添加选项按钮** — 使用"创建对话框按钮"动作，依次创建"音量设置"、"难度选择"、"开始游戏"三个按钮[^1]。每个按钮需要设置显示文字。

4. **第四步：编写响应触发器** — 创建新触发器，添加事件"对话框 - 对话框按钮被点击"。然后用条件判断是哪个按钮被点击，分别执行对应的操作[^2]。

> **💡 新手提示**：按钮较多时，建议使用"数组"变量存储按钮，这样可以用循环批量创建和管理。数组就像一排编好号的抽屉，每个按钮对应一个编号。

> **⚠️ 常见错误**：忘记在玩家点击按钮后隐藏对话框。对话框不会自动消失，必须用"隐藏对话框"动作手动关闭，否则会一直显示在屏幕上。

### 制作确认/取消对话框

确认对话框常用于重要操作前的二次确认，比如"确认退出游戏"或"确认购买物品"。

1. **第一步：创建对话框** — 新建一个触发器，创建对话框并设置标题为"确认操作"。

2. **第二步：创建两个按钮** — 创建"确定"和"取消"两个按钮[^1]。按钮文字要清晰明确，让玩家一眼就知道点击后的后果。

3. **第三步：设置按钮位置** — 取消按钮建议放在确定按钮下方或右侧。可以通过设置偏移值(X、Y坐标)来调整位置[^1]。

4. **第四步：编写按钮响应** — 在触发器中使用条件"[对话框按钮]等于[确定按钮]"来判断玩家选择。确定时执行操作，取消时可用"隐藏对话框"动作关闭菜单[^3]。

> **💡 新手提示**：如果同一个对话框要在多处使用，可以把"创建对话框"放在初始化触发器中，只创建一次，然后用"显示/隐藏对话框"动作控制显示状态，避免重复创建。

> **⚠️ 常见错误**：在按钮点击触发器中使用"等待"动作。对话框事件响应中禁止使用等待，否则可能导致按钮无响应或卡顿。

### 多选项对话框示例

多选项对话框最经典的用途是英雄选择界面。

1. **第一步：规划选项数量** — 决定要提供几个选项，比如4个英雄供玩家选择。

2. **第二步：创建对话框** — 设置标题为"选择你的英雄"。

3. **第三步：创建多个按钮** — 使用"数组变量"存储按钮。为每个按钮设置不同文字，如"战士"、"法师"、"刺客"、"牧师"[^1]。

4. **第四步：设置切换逻辑** — 如果选项太多一页显示不下，可以创建多个对话框。第一页显示4个选项，点击"下一页"按钮后隐藏第一页，显示第二页[^2]。

5. **第五步：保存选择结果** — 当玩家选择后，用变量记录选择的英雄类型，然后创建对应的英雄单位给玩家。

> **💡 新手提示**：使用"对话框 - 显示/隐藏"动作时，要指定对哪个玩家显示。推荐使用"触发玩家"变量，这样每个玩家只能看到自己的对话框[^3]。

> **⚠️ 常见错误**：忘记为每个玩家单独显示对话框。如果不指定玩家，对话框可能显示错误或所有玩家看到同一个对话框。

### 小结

完成以上三个示例后，你已经掌握了对话框的核心用法。现在你可以尝试制作一个完整的英雄选择界面：创建对话框 → 添加多个英雄按钮 → 点击后创建对应英雄 → 隐藏选择菜单。建议从简单的"确认/取消"对话框开始练习，再逐步挑战复杂的多选项菜单。
