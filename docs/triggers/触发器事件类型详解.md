# 触发器事件类型详解

## 触发器事件类型概述

本节我们将学习什么是"触发器"（Trigger），以及触发器的基本结构。触发器是魔兽争霸3地图的灵魂——它能让你的地图"活"起来，比如"当玩家点击按钮时，给英雄加100金币"这样的逻辑，都靠触发器实现。

### 触发器基本结构（事件-条件-动作）

触发器就像一个"如果...那么..."的机器，它由三部分组成：**事件（Event）**、**条件（Condition）**、**动作（Action）**[^1]。

> **💡 新手提示**：可以把触发器想象成一道"如果...就..."的数学题：
> - **事件**：如果发生什么事...
> - **条件**：并且满足什么条件...
> - **动作**：那么就执行什么操作

### 事件在触发器中的作用

**事件**是触发器的"开关"——没有事件，触发器永远不会运行。你可以理解为：事件就是"告诉触发器'什么时候开始工作'"。常见的事件包括[^1][^2]：

| 事件类型 | 含义 |
|---------|------|
| 单位死亡 | 当某个单位死亡时 |
| 单位进入区域 | 当单位走进某个区域时 |
| 玩家聊天 | 当玩家输入聊天信息时 |
| 定时器到期 | 每隔一段时间 |

### 小结

完成以上内容后，你应该理解了：
- 触发器由**事件-条件-动作**三部分组成
- **事件**决定了触发器"何时启动"
- 不同的**事件类型**对应不同的游戏情况

> **⚠️ 常见错误**：很多新手会混淆"条件"和"动作"。记住：**条件**是"检查一下"，**动作**是"真的去做"。条件可以没有，但事件必须有，否则触发器永远不会运行！

## 地图初始化与游戏开始事件

在本节中，你将学习如何创建**地图初始化事件**和**游戏开始事件**——这是几乎所有RPG地图最先执行的两个核心事件。学会它们之后，你就能在玩家进入游戏的第一时间设置变量、创建单位、显示欢迎信息等功能[^1]。

### 地图加载事件

地图加载事件（又叫"地图初始化"）会在地图完全加载时自动触发，不需要玩家做任何操作。这是最适合做**准备工作**的事件，比如初始化变量（存放数据的容器）、预加载资源等。

#### 操作步骤

1. **打开触发器编辑器** — 在World Editor顶部菜单栏点击"触发器"按钮（或者按快捷键F4）[^1]
2. **新建一个触发器** — 在左侧面板空白处右键，选择"新建触发器"，将它改名为"初始化设置"
3. **添加事件** — 点击触发器中的"事件"区域，在弹出的列表中找到"地图初始化"并点击选择
4. **添加动作** — 点击"动作"区域，从左侧分类中找到"游戏-显示教程提示"（这只是测试用，实际项目中你会添加更多初始化动作）

> **💡 新手提示**：地图初始化事件会在**任何玩家**进入地图之前就触发，所以如果你想设置只有某个玩家能看到的UI，需要使用"游戏开始事件"而不是这里的事件。

> **⚠️ 常见错误**：新手经常把初始化代码写在错误的位置（比如写在"单位死亡"事件里），导致游戏开始时变量还没准备好。记住：所有**准备工作**（设置初始变量、创建起始单位等）都应该放在"地图初始化"或"游戏开始"事件下。

### 游戏开始事件的使用场景

"游戏开始事件"与"地图初始化"非常相似，但有一个关键区别：它会在**所有玩家都进入地图后**才触发。这对于需要等待所有玩家准备就绪的场景特别有用[^1]。

#### 适合使用游戏开始事件的场景

- 显示开场动画或过场文字
- 为每个玩家创建专属单位（如主角）
- 分配玩家的起始装备或金币
- 启动计时器或回合系统

#### 操作步骤

1. 在触发器编辑器中新建一个触发器，命名为"游戏开始设置"
2. 在事件区域点击"新事件"按钮
3. 在搜索框中输入"游戏开始"，从结果中选择"游戏 - 游戏开始时"
4. 现在你可以添加具体的动作了，比如"游戏 - 显示教程提示"设置为"欢迎来到口袋妖怪RPG！"

> **💡 新手提示**：如果你不确定该用哪个事件，记住这个简单规则——需要所有玩家都就位才能执行的功能，用"游戏开始"；只需要地图加载好就能执行的准备操作，用"地图初始化"。

### 小结

完成以上步骤后，你已经掌握了：
- 如何创建**地图初始化事件**并添加动作
- 如何创建**游戏开始事件**并设置玩家专属内容
- 两个事件的区别以及何时该用哪一个

这两个事件是所有RPG地图的"根基"，建议你在制作任何地图时都先创建这两个触发器，把初始化代码整理好放在里面。

## 玩家相关事件

在本节中，你将学习如何监听玩家的进出游戏状态以及聊天消息。掌握这些事件后，你就可以制作登录欢迎系统、聊天指令功能，甚至简单的文字RPG指令系统。

### 操作步骤

1. **打开触发器编辑器** — 在World Editor（世界编辑器）界面，按 `F4` 键打开触发器编辑器窗口[^1]。如果没有触发器，可右键点击左侧列表，选择"新建触发器"创建。

2. **创建玩家进入事件** — 展开"事件"分类，选择"单位 - A unit enters playable area"（单位进入可玩区域）。这个事件会在任意玩家的单位进入地图时触发，非常适合用来检测玩家加入游戏[^1]。

3. **创建玩家离开事件** — 在事件列表中选择"玩家 - Player leaves game"（玩家离开游戏）。设置你想监听的具体玩家编号（0-23，对应红、绿、蓝等颜色）。

4. **创建聊天事件** — 选择"玩家 - Player Chat Message"（玩家聊天信息）。在这里你可以设置：
   - 聊天前缀（如 "/" 或 "!"）
   - 是否仅检测不含前缀的普通聊天

> **💡 新手提示**：在设置玩家聊天事件时，"Triggering Player"（触发玩家）是一个非常重要的变量，它能帮你判断是谁发送了这条命令。

> **⚠️ 常见错误**：新手经常把"单位进入可玩区域"和"玩家离开游戏"搞混。前者需要单位对象，后者只需要玩家编号。如果你想做登录欢迎界面，应该用玩家离开/进入游戏事件。

### 小结

完成以上步骤后，你应该已经掌握了三种最常用的玩家相关事件：
- 单位进入可玩区域（适合检测玩家进入）
- 玩家离开游戏（适合做离开提示）
- 玩家聊天信息（适合做指令系统）

你可以尝试组合这些事件，比如当玩家聊天输入 "/help" 时显示帮助信息，这正是很多RPG游戏指令系统的基础[^1]。

## 单位事件详解

本节将详细介绍 Warcraft III 触发器中最常用的**单位事件**类型。学完本节后，你将能够创建"当某个单位出生时触发"、"当单位受伤时触发"等功能，让你的地图具有更强的交互性。

### 单位出生与死亡事件

单位出生与死亡事件是 RPG 地图中最基础的事件类型，常用于任务系统、奖励发放或怪物刷新。

#### 操作步骤

1. **打开触发编辑器** — 在 World Editor 界面顶部菜单栏点击「触发器」按钮（或者按 F4 快捷键）[^1]
2. **创建新触发器** — 右键左侧触发器列表，选择「新建触发器」，命名为"单位死亡检测"
3. **添加事件** — 点击触发器右上角的「新增事件」按钮，在弹出窗口中选择「单位 - 单位死亡」[^2]
4. **设置条件（可选）** — 添加「单位 - 单位属于敌对单位」等条件来过滤不需要的死亡事件
5. **添加动作** — 添加「游戏 - 显示教程提示」等动作来测试效果

> **💡 新手提示**：单位死亡事件只能检测"真正死亡"的情况，如果单位被放逐（Banish）或变形，不会触发此事件。

> **⚠️ 常见错误**：新手经常把「单位复活」事件和「单位出生」事件混淆。「单位出生」只在完全新建单位时触发，复活单位请使用「单位 - 单位进入区域」事件。

### 单位被攻击与受伤事件

这类事件让你可以监听单位的战斗状态，用于制作反击系统、伤害显示或闪避判定。

#### 操作步骤

1. **新建触发器** — 命名为"受伤检测"
2. **添加事件** — 选择「单位 - 单位受到攻击」或「单位 - 单位受到伤害」[^2]
3. **获取受伤单位** — 从事件函数中选择「受到攻击的单位」或「受到伤害的单位」
4. **创建变量** — 新建「受伤单位」变量，类型为「单位」，用于存储事件触发的单位
5. **添加动作示例** — 比如播放受伤音效、显示伤害数字或触发反击逻辑

> **💡 新手提示**：「受到攻击」和「受到伤害」的区别在于：攻击是攻击动作开始时触发，伤害是实际扣血时触发。如果单位有护盾或闪避技能，两者触发时机可能不同。

> **⚠️ 常见错误**：不要在受伤事件中无限循环添加buff，这会导致游戏卡顿。每次受伤最多添加一次增益效果，建议配合「布尔变量」来控制。

### 单位获得/失去增益事件

增益（Buff）事件用于监听魔法效果的施加与移除，是技能系统、自动治疗等机制的基础[^3]。

#### 操作步骤

1. **新建触发器** — 命名为"Buff监听"
2. **添加事件** — 选择「单位 - 单位获得增益」或「单位 - 单位失去增益」
3. **获取增益类型** — 从事件函数中选择「被施放的技能」来判断是什么buff
4. **条件设置** — 添加条件判断，如「技能等于 圣光术」
5. **添加对应动作** — 例如获得治疗buff时添加回血动作，失去buff时移除相关效果

> **💡 新手提示**：学会善用「拥有技能/拥有buff」条件检测。你可以用这个制作"中毒时移动速度降低"或"获得隐身buff后隐藏单位"等功能。

> **⚠️ 常见错误**：新手容易忽略「失去增益」事件的处理。如果没有正确移除对应的视觉效果或逻辑状态，单位可能会出现"假死"或"永久隐身"的bug。

### 小结

完成以上三个单位事件类型的学习后，你应该掌握了：
- 如何创建检测单位死亡并触发相应动作的触发器
- 如何获取受伤/被攻击的单位信息并进行逻辑处理
- 如何监听单位获得或失去特定buff并做出响应

这些基础事件组合使用，可以构建出大多数 RPG 地图的核心战斗系统。建议下一步尝试制作一个"怪物死亡后掉落金币"的简单功能来巩固所学知识。

## 时间与游戏进程事件

本节将教你两种最常用的自动触发事件：**周期性时间事件**（每隔一段时间自动执行）和**游戏时间事件**（当游戏时间达到某个值时执行）。学完本节，你将能够创建"每10秒生成一个怪物"或"游戏进行30分钟后显示胜利提示"这样的功能。

### 周期性时间事件

1. **打开触发编辑器** — 在顶部菜单栏点击"触发器"按钮，打开触发编辑器窗口[^1]

2. **选中或新建触发器** — 在左侧触发器列表中，点击选择你要编辑的触发器（或者右键新建一个）

3. **添加事件** — 在触发器编辑器中，点击"事件"区域，然后点击右侧的"新建事件"按钮

4. **选择时间事件类型** — 在事件分类中找到"时间"类别，选择"周期性事件"下的"每N秒后"选项[^2]

5. **设置间隔时间** — 在弹出的对话框中输入数字。例如输入"10"表示每隔10秒触发一次；输入"0.5"表示每0.5秒（每半秒）触发一次

6. **完成添加** — 点击"确定"，事件就添加成功了

> **💡 新手提示**：周期性事件的间隔时间可以是小数！如果想每2.5秒触发一次，直接输入"2.5"即可。

> **⚠️ 常见错误**：新手容易把间隔时间设为"0"，这会导致触发器每秒执行极多次，可能导致游戏严重卡顿甚至崩溃。最小建议设置为"0.1"以上。

### 游戏时间与回合事件

1. **继续在事件区域操作** — 在同一个触发器中（或新建另一个触发器），再次点击"事件"区域

2. **新建事件** — 点击"新建事件"按钮

3. **选择游戏时间事件** — 在"时间"分类下，选择" elapsed game time equal to"（已流逝游戏时间等于）选项[^1]

4. **设置触发时间点** — 输入触发时刻（以秒为单位）。例如输入"180"表示游戏时间达到3分钟（180秒）时触发

5. **添加对应动作** — 设置当游戏到达该时间时要执行的操作，比如"显示胜利消息"或"触发结局剧情"

> **💡 新手提示**：游戏时间事件只会触发**一次**——当时间刚到达你设定的值时执行。如果你想让某个功能"每分钟"重复，应该用周期性事件而非游戏时间事件。

> **⚠️ 常见错误**：混淆"游戏时间"和"真实时间"。Warcraft III的"游戏时间"是从游戏开始算起的内部计时器，不受暂停影响。暂停游戏时游戏时间不会前进。

### 小结

完成以上步骤后，你应该掌握了：
- 如何添加**周期性事件**（每隔X秒自动执行）
- 如何添加**游戏时间事件**（当游戏进行到X秒时执行一次）
- 周期性事件适合循环任务（如刷怪、恢复生命）
- 游戏时间事件适合一次性剧情或阶段切换（如BOSS战、结局）

你可以尝试创建一个"每15秒在地图中央生成一个野生怪物"的触发器，这是RPG地图中非常常见的机制！

## 其他常用事件类型

本节将介绍两组实用的事件类型：**可破坏物与物品事件**和**对话框事件**。学完后，你能让玩家拾取物品时获得奖励，或让玩家点击按钮时触发特定功能，比如打开商店、切换模式等。

### 可破坏物与物品事件

1. **添加"单位获得物品"事件** — 在触发器编辑器中，点击事件旁边的绿色"新建"按钮[^1]，在弹出的列表中找到"单位 - A unit Acquires an item"（单位获得物品）并点击选中。这个事件会在任何单位（玩家单位或中立敌对单位）获得物品时触发。
2. **添加"单位失去物品"事件** — 同样方式新建事件，选择"A unit Loses an item"（单位失去物品）。常用于检测玩家是否丢弃了某个关键道具。
3. **使用物品条件过滤** — 在触发器的"条件"区域，添加"Unit - Unit of type"条件，并指定你想要的物品类型（如"生命药水"）。这样只有获得特定物品时才会执行后面的操作[^2]。

> **💡 新手提示**：物品事件的"单位"指的是持有物品的英雄或士兵。地图上的物品（如木箱中的装备）被拾取后，才会触发"获得物品"事件。

> **⚠️ 常见错误**：新手容易混淆"获得"和"拾取"——"拾取"事件只对有背包栏位的单位有效，而"获得"包括系统自动给予物品的情况。

### 对话框与变量事件

1. **创建对话框** — 在触发器中新建动作，选择"Dialog - Clear"或"Dialog - Create from button"。对话框是一个隐藏的界面元素，不会自动显示，需要用动作手动显示给玩家[^3]。
2. **添加对话框按钮** — 新建动作"Dialog - Create a button for Dialog"，设置按钮文字（如"开始游戏"、"退出"）。每个按钮都可以分配一个"Clicked Button"事件。
3. **绑定对话框点击事件** — 新建事件，选择"Dialog - A dialog button is clicked"。这是对话框系统的核心事件——当玩家点击任意对话框按钮时触发，触发器会自动记录被点击的是哪个按钮。
4. **使用变量存储对话框** — 把对话框和按钮保存到变量中，这样可以在多个触发器里重复使用同一个对话框[^2]。

> **💡 新手提示**：对话框事件需要先"显示对话框"给玩家看，否则玩家无法点击。常用动作是"Dialog - Show all dialogs for Player"。

> **⚠️ 常见错误**：忘记把对话框保存到变量中！如果你在触发器A创建了对话框但在触发器B才引用它，必须用变量存储，否则触发器B找不到这个对话框。

### 小结

完成以上步骤后，你应该已经掌握了物品事件的添加与条件过滤，以及对话框的创建、按钮配置和点击事件。这些事件组合起来，可以实现完整的物品奖励系统和玩家交互菜单——这是制作RPG地图不可或缺的两种功能。
