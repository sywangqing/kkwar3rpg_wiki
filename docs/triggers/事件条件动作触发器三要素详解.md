# 事件、条件、动作：触发器三要素详解

## 概述

这一节我们要学习触发器的**三个核心要素**：事件、条件、动作。学完这节后，你会理解触发器是如何工作的，以及为什么某些功能"不生效"——这往往是新手最容易遇到的问题。

### 什么是触发器？

**触发器**（Trigger），简单理解就是游戏里的"自动开关"。你可以把它想象成一条规则：**"当某件事发生时（事件），如果满足某个条件（条件），就执行某个动作（动作）"**。

打个比方：当你的角色走进一扇门（事件），如果身上有钥匙（条件），门就会打开（动作）——这就是一个完整的触发器逻辑。[^2]

> **💡 新手提示**：触发器是Y3编辑器里最核心的功能模块，几乎所有游戏逻辑都要靠它来实现。掌握好三要素，就等于掌握了编辑器最重要的技能。

### 操作步骤

1. **第一步：打开触发器编辑器** — 在顶部菜单栏找到"触发器"选项并点击进入[^1]
2. **第二步：观察默认界面** — 你会看到左侧是触发器列表，右侧是编辑区域，分为"事件"、"条件"、"动作"三个部分[^2]
3. **第三步：理解三者关系** — 记住这个顺序：**事件触发 → 检查条件 → 执行动作**，缺一不可[^3]

### 小结

完成以上步骤后，你应该已经理解：触发器由"事件"、"条件"、"动作"三部分组成，只有当事件发生**并且**条件满足时，动作才会执行。下一节我们将学习如何亲手创建一个触发器！
