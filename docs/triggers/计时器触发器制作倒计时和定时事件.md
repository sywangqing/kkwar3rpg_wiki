# 计时器触发器：制作倒计时和定时事件

## 什么是计时器触发器

计时器触发器是一种特殊的触发器（Trigger），它不是等待"某个事件发生"才执行，而是**按照设定的时间自动执行**。简单来说，普通触发器是"等事件来敲门"，而计时器触发器是"自己按时敲门"。学会使用计时器触发器后，你可以制作倒计时游戏、回合制系统、波次刷怪（每隔一段时间自动生成敌人）等有趣的功能[^6][^8]。

### 计时器触发器的作用与使用场景

计时器触发器最常见的用途包括：

1. **倒计时系统** — 比如比赛开始前的倒计时显示，或者道具持续时间的倒计时显示[^8]
2. **波次生成** — 每隔30秒自动生成一波敌人，这就是RPG地图中常见的"刷怪波次"功能[^6]
3. **定时效果** — 每隔一段时间给玩家加一次血量或buff（增益效果）
4. **游戏时间追踪** — 记录游戏已经进行了多久[^8]

> **💡 新手提示**：计时器触发器就像是你的"闹钟"，你设定好时间，它就会在时间到了时提醒你执行某个动作。

### 计时器触发器 vs 普通事件触发器的区别

| 对比项 | 普通事件触发器 | 计时器触发器 |
|--------|---------------|-------------|
| 触发方式 | 等待特定事件发生（如单位进入区域、拾取物品）[^2][^3] | 按照设定的时间间隔自动执行 |
| 典型场景 | "玩家进入这个区域 → 触发剧情" | "每30秒 → 刷一波怪物" |
| 持续性 | 通常执行一次就结束 | 可以循环执行（反复触发） |

**普通事件触发器**像是一个"被动技能"——必须有人做某件事才会触发；而**计时器触发器**是一个"主动技能"——它会按照你的时间表主动运行，即使没有任何玩家操作也不会停止。

> **⚠️ 常见错误**：新手容易混淆"等待（Wait）"和"计时器（Timer）"。等待（Wait）会让整个触发器暂停，期间无法做其他事；而计时器是独立运行的，不会卡住其他代码[^7]。如果你需要同时做多件事，用计时器而不是等待！

### 小结

通过本节，你已经了解了：
- **计时器触发器**是一种按时间自动执行的触发器
- 它适合制作倒计时、波次刷怪、定时效果等系统
- 它与普通事件触发器的核心区别在于"主动执行"vs"被动响应"

下一节我们将学习如何**创建和配置计时器触发器**，敬请期待！

## 计时器的两种类型

在接下来的内容中，我们将学习计时器的两种类型。完成本节学习后，你将能够根据不同的游戏需求，选择合适的计时器类型来控制游戏事件的执行时机。

### 单次计时器（只触发一次）

**单次计时器**就像厨房里用的计时器——设定好时间后，它只会响一次，然后停止[^6]。当你需要某个事件**只发生一次**时，就使用这种计时器。比如：
- 游戏开始前的倒计时
- 玩家进入传送门后，5秒后把他送到另一个地点
- BOSS死亡后，10秒后显示胜利画面

单次计时器的特点是：触发一次后自动结束，不会再次运行。

### 重复计时器（循环触发）

**重复计时器**就像闹钟的"循环闹铃"——设定好间隔后，它会**每隔一段时间触发一次**，周而复始[^6]。当需要游戏事件**反复循环执行**时，就使用这种计时器。比如：
- 每隔30秒生成一波敌人
- 每10秒给所有玩家发放金币
- 每60秒切换一次昼夜

重复计时器的特点是：会无限循环，除非你手动停止它。

### 何时使用哪种计时器

选择哪种计时器，只需要问自己一个问题：**这个事件需要重复执行吗？**

- 如果事件**只执行一次** → 选择单次计时器
- 如果事件**需要定期循环** → 选择重复计时器

> **💡 新手提示**：在设计游戏逻辑前，先想清楚事件的发生次数，这能帮你避免后续的修改麻烦。比如"游戏结束"事件只能用单次计时器，而"敌人刷新"必须用重复计时器。

> **⚠️ 常见错误**：新手经常搞混两种计时器的用途。比如想做"每分钟给玩家发金币"，却用了单次计时器，导致只发了一次就停止了。解决方法是在创建计时器时，仔细看清楚是"单次"还是"重复"

## 创建倒计时触发器的基本步骤

本节将教你如何在 Warcraft 3 世界编辑器中创建最基本的倒计时触发器。学完本节后，你就能让游戏中的某个事件在指定时间后自动执行——比如30秒后敌人出现，或者5分钟后开启特殊模式。

### 创建计时器动作

1. **打开触发编辑器** — 在世界编辑器主界面点击顶部菜单的"触发器"按钮[^7]
2. **新建触发器** — 在左侧触发器列表空白处右键，选择"新建触发器"
3. **重命名触发器** — 双击触发器名称，改为容易识别的名字，比如"30秒倒计时"
4. **添加"启动计时器"动作** — 在触发器编辑窗口中，点击右侧"新建动作"按钮，在搜索框输入"计时器"，找到"启动计时器"并点击添加[^6]

### 设置计时器等待时间

1. **配置计时器参数** — 选中刚添加的"启动计时器"动作，在底部属性面板设置以下参数：
   - **计时器时间**：输入你想要的秒数（如 30.00 表示30秒）
   - **重复模式**：选择"是"让计时器循环，还是"否"只执行一次[^8]
2. **添加"等待"动作（可选）** — 如果你只想简单延迟一下，可以添加"等待"动作，设置等待时间为 10.00 秒[^7]

> **💡 新手提示**：计时器时间建议用小数，比如 30.00 而不是 30，这样计时更精确。另外，"启动计时器"动作会在后台运行，不阻塞游戏进行。

### 将计时器绑定到触发器

1. **添加触发事件** — 在触发器中新建事件，选择"计时器"类别下的"计时器过期"事件[^5]
2. **编写触发结果** — 在"事件"下方添加你想执行的动作，比如"创建单位"或"显示对话框"[^6]
3. **测试计时器** — 按 F6 打开游戏测试，观察计时器是否在指定时间后触发

> **⚠️ 常见错误**：新手经常忘记添加"计时器过期"事件，导致计时器启动了但没有任何反应。解决方法：确保你的触发器同时包含"启动计时器"动作（用于启动计时器）和"计时器过期"事件（用于响应计时结束）。

### 小结

完成以上三个步骤后，你应该已经创建了一个可以正常工作的倒计时触发器。当你进入游戏测试时，系统会在你设置的时间（比如30秒）后自动执行你指定的命令。你可以复制这个触发器并修改时间，创造出多个不同时间点的游戏事件，比如"第一波敌人30秒后出现"、"60秒后开启商店"、"90秒后Boss登场"等复杂玩法[^5][^8]。

## 在屏幕上显示倒计时

本节我们将学习如何在游戏屏幕上实时显示倒计时，让玩家能够清楚地看到剩余时间。学会后，你可以为回合制游戏添加回合限制，或者为限时活动添加紧迫感。

### 操作步骤

1. **创建计时器变量** — 在触发器编辑器的左侧，找到"变量"面板，点击📦新建变量，给变量起名为"倒计时计时器"，类型选择"计时器"[^5]。

2. **初始化计时器** — 新建一个触发器命名为"初始化倒计时"，在"地图初始化"触发器中调用它。添加动作：计时器 - 启动倒计时计时器（设置为30秒，重复：不重复）[^6]。

3. **创建显示用的多边形文本** — 在触发器中添加动作：游戏 - 显示文本消息（设置为"剩余时间：" + 对计时器取整），但这样会一直弹消息，不够美观。

4. **使用计时器窗口显示倒计时** — 更好的方法是：添加动作"计时器 - 创建计时器窗口"，将计时器绑定到屏幕上方的计时条[^5]。这样玩家可以看到一个漂亮的进度条在倒数。

5. **设置实时更新触发器** — 创建一个新的触发器，事件选择"每X秒"（建议0.5秒或1秒），动作里更新计时器窗口显示的数字[^7]。

> **💡 新手提示**：计时器窗口会自动显示时间，但你也可以同时用"游戏 - 显示文本消息"作为辅助提示，让倒计时更醒目。

> **⚠️ 常见错误**：很多新手直接用"等待"动作来倒计时，但这样会阻塞其他触发器。正确做法是用计时器事件来驱动倒计时逻辑[^7]。

### 小结

完成以上步骤后，你的游戏界面顶部会出现一个实时更新的倒计时条，玩家能清楚地看到剩余时间。当倒计时归零时，记得添加一个"游戏 - 显示胜利/失败信息"的提示，告知玩家时间已到。

## 定时事件的实际应用示例

本节将通过三个具体的游戏功能案例，手把手教你如何使用计时器触发器。学会后，你就能为地图添加怪物刷新、技能冷却、游戏阶段切换等常见功能。

### 怪物刷新倒计时

这是RPG地图中最常用的功能之一。当一波怪物被击杀后，计时器开始倒计时，倒计时结束后自动刷新下一波怪物。[^6]

1. **第一步：创建计时器变量** — 在触发编辑器左侧的变量列表（Variable List）区域，点击"新建变量"按钮[^5]。将变量命名为"怪物刷新计时器"，类型选择"计时器"（Timer）

2. **第二步：创建刷新触发器** — 在触发器列表中新建一个触发器，命名为"波次刷新"。事件设为"计时器 过期"（Timer - Expires），条件选择刚才创建的"怪物刷新计时器"[^6]

3. **第三步：添加刷新动作** — 在响应动作中添加创建单位组的行为。比如使用"选择所有敌人单位"再"移除已选中单位"，然后"在指定点创建新的怪物单位"。完成后，再重新启动计时器，为下一波做准备[^6]

> **💡 新手提示**：设置计时器时，记得选择"倒计时"类型而不是"已用时间"。倒计时是从设定值往下减，适合显示给玩家看。

> **⚠️ 常见错误**：忘记在刷新完成后重新启动计时器。这会导致第一波刷新后，后续波次就再也不刷新了。解决方法是记得添加"启动计时器"动作。

### 技能冷却定时器

技能冷却是指玩家使用技能后，该技能会进入冷却状态，在冷却期间无法再次使用。[^2]

1. **第一步：创建冷却计时器变量** — 新建一个计时器变量，命名为"技能A冷却计时器"。再建一个实数变量"技能冷却时间"，设为3.0（即3秒）

2. **第二步：设置技能释放触发器** — 创建一个触发器，事件为"单位 使用技能"。动作中先"启动单次计时器"，时间为"技能冷却时间"变量，设置为"只执行一次"[^7]

3. **第三步：冷却结束响应** — 新建一个触发器，事件为"计时器 过期"，条件检查是否为"技能A冷却计时器"。动作中可以使用"禁用技能按钮"配合"等待"再"启用技能按钮"，或者直接通过变量控制技能能否使用

> **💡 新手提示**：如果想让多个技能共享冷却，可以共用同一个计时器。如果各自独立冷却，就需要分别创建不同的计时器变量。

### 游戏阶段转换定时

有些地图会有"白天/黑夜"循环，或者"准备阶段→战斗阶段→结算阶段"这样的流程切换。

1. **第一步：创建阶段变量** — 新建整数变量"当前游戏阶段"，初始值设为1。再创建一个计时器变量"阶段切换计时器"

2. **第二步：初始化第一阶段** — 在地图初始化触发器中，添加"启动计时器"，时间设为60秒（或其他需要的时长），类型为"倒计时"[^5]

3. **第三步：阶段切换逻辑** — 创建触发器响应计时器过期，动作中先将"当前游戏阶段"变量加1，然后根据新的阶段值执行对应操作：阶段2时开放战斗区域，阶段3时显示结算面板。最后重新启动计时器进入下一个循环[^5]

## 常见问题与优化技巧

本节将帮助你解决使用计时器时最常遇到的三个问题：资源泄漏、多计时器管理和精度问题。学完本节后，你的地图将更加稳定，不会出现卡顿或计时不准的情况。

### 操作步骤

1. **理解什么是资源泄漏** — 计时器和单位、物品一样，是一种"资源"。当你创建一个计时器时，计算机会占用一小块内存。如果创建后不用了却不销毁，这块内存就会一直占用着，导致游戏越来越慢，最终可能崩溃。[^7]

2. **养成销毁计时器的好习惯** — 每次用完计时器，必须用"Destroy Timer"（销毁计时器）动作把它删掉。就像用完杯子要洗一样，用完计时器要销毁。[^5]

3. **使用变量存储每个计时器** — 当你需要同时运行多个计时器时（比如游戏中有多个倒计时），一定要把每个计时器存到单独的变量里。这样才能区分和单独控制每一个计时器。[^8]

4. **选择正确的时间类型** — 计时器有两种时间模式："游戏时间"和"真实时间"。游戏时间在暂停时会停止，真实时间则按实际流逝计算。根据你的需求选择合适的模式——比如显示给玩家的倒计时，建议用真实时间，这样即使暂停也不会乱。

> **💡 新手提示**：每次启动计时器时，同一个计时器变量只能保存一个计时器。如果给已存在的计时器变量赋值新计时器，旧的计时器引用就会丢失且无法销毁，造成泄漏！

> **⚠️ 常见错误**：很多新手以为Wait（等待）动作也能做倒计时。但Wait会"卡住"整个触发器，期间这个触发器什么都不能做。正确做法是用计时器配合"到期事件"，让系统自动在时间到了时执行代码。[^6]

### 小结

完成以上步骤后，你的计时器系统应该已经非常稳定了。记住三个关键原则：① 用完必须销毁，② 多个计时器用变量管理，③ 能用计时器就不用Wait。坚持这样做，你的地图就不会出现计时器相关的卡顿问题了。
