# 触发器系统概述

## 概述

触发器系统是魔兽地图编辑器的"大脑"——它决定了游戏中的**一切事件**该如何响应。比如当一个单位死亡时显示什么信息，当玩家建造建筑时扣除多少资源，当游戏开始时显示什么提示……这些都需要触发器来完成。

简单来说，**触发器 = 事件 + 条件 + 动作**。当某个"事件"发生时，系统会检查"条件"是否满足，如果满足就执行预设的"动作"。比如："当单位死亡（事件）→ 如果是英雄（条件）→ 显示失败画面（动作）"。

### 为什么必须学习触发器？

如果你只是摆放地图地形和单位，地图会像一个没有规则的沙盒——单位不会移动，技能不会生效，建筑不会产出资源。**触发器让地图"活"起来**，是制作RPG、PVP、TD等各类地图必不可少的功能。

### 学习目标

学完本节后，你将：

- 理解触发器的基本结构和三大组成部分
- 能够在触发器编辑器中创建第一个触发器
- 理解什么是事件、条件和动作

> **💡 新手提示**：不要被"触发器"这个名字吓到！它本质上是"如果…那么…"的逻辑。你每天都在做这样的判断，只是现在要用电脑语言来表达而已。

> **⚠️ 常见错误**：新手常常跳过触发器编辑器，只做地形和单位装饰，然后奇怪为什么地图玩不起来。记住——**不写触发器的地图只是静态画面，不是游戏**。
