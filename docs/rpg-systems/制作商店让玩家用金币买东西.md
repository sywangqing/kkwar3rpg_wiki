# 制作商店：让玩家用金币买东西

## 什么是商店系统

本节将带你认识什么是"商店系统"，以及它在RPG游戏中为什么如此重要。学完本节后，你将理解商店的基本原理，为后续创建可交互的购买系统做好准备。

### 商店在RPG中的作用

在大多数RPG游戏中，玩家通过完成任务或击败怪物来获取**金币**。然而，如果玩家辛苦赚来的金币无处可花，游戏就会失去一个重要的成就感来源。商店系统正是解决这个问题的关键——它为玩家提供了一个用金币换取装备和道具的地方[^1]。

想象一下，玩家刚刚经历了一场艰难的战斗，获得了500金币。然后他走到一个铁匠铺，用这些金币购买了一把更强的剑——这种"付出→回报→成长"的循环，正是RPG游戏让人上瘾的核心机制之一。

### 商店单位的两种类型

在魔兽争霸3地图编辑器中，商店实际上也是一种特殊的**单位**。编辑器内置了两种主要的商店模板[^2]：

**第一种：背包商店（Shop）**
- 这种商店自带一个物品展示界面
- 玩家点击商店后，会自动弹出购买窗口
- 适合销售消耗品、药水等常规物品

**第二种：仓库商店（Full Shop）**
- 同样是自带的界面
- 但可以放置更多种类的物品
- 适合需要大量商品的大型商店

> **💡 新手提示**：在编辑器左侧的"单位"面板底部，有一个专门的"商店"分类，里面列出了所有可用的商店模板[^3]。你可以直接把这些模板拖拽到地图上使用。

> **⚠️ 常见错误**：新手常常忘记商店需要"是商店单位"这个属性设置。在单位的"标签（Flags）"中，必须勾选"商店"选项，否则玩家即使点击了建筑，也看不到任何购买界面[^1]。

### 小结

现在你已经理解了商店系统的基本概念：
- 商店是一种让玩家用金币购买物品的特殊单位
- 编辑器提供了两种内置商店模板
- 正确的属性设置是让商店正常工作的前提

**下一步预告**：了解了基本概念后，我们将开始实际操作——创建第一个属于自己的商店单位！

## 创建和放置商店单位

在本节中，我们将学习如何在地图上创建商店单位，并让它显示在玩家面前。学完本节后，你将拥有一个可以供玩家购买物品的商店！

### 操作步骤

1. **打开对象管理器** — 在顶部菜单栏中点击"地图"菜单，然后选择"对象管理器"[^3]。对象管理器就像是游戏的"物品清单"，里面包含了地图中所有单位、物品和技能的定义。

2. **找到商店单位** — 在对象管理器的左侧面板中，点击"单位"选项卡。然后在搜索框中输入"商店"，你就能看到多种商店单位[^1]。不同的商店外形不同，但功能是一样的——都允许玩家购买物品。选择一个你喜欢的商店，比如"娜萨的物品商店"。

3. **将商店拖放到地图上** — 选中对象管理器中的商店单位，然后切换到地形编辑器视图。在你想要放置商店的位置点击地图，商店单位就会出现[^4]。建议把商店放在地图中央或者玩家出生点附近，方便玩家找到。

4. **设置商店的基本属性** — 双击地图上的商店单位，弹出"单位属性"窗口。在这里你可以修改：
   - **名称**：改成你想要的商店名字
   - **皮肤/模型**：可以更换商店的外观
   - **所属玩家**：设置为玩家可控制（通常设为1号玩家或中立）

> **💡 新手提示**：如果你在对象管理器中找不到"商店"关键字，试试搜索英文"shop"，因为有些翻译不统一。

> **⚠️ 常见错误**：新手经常忘记设置"所属玩家"属性，导致商店变成"不可控制"状态。解决方法是将所属玩家改为"1"（1号玩家）或"12"（中立敌对）。

### 小结

完成以上步骤后，你应该能在地图上看到你的商店单位了！但现在商店还是空的，玩家进去什么都买不到。下一节我们将学习如何给商店添加可购买的物品。

## 配置商店出售的物品

本节将教你如何在地图编辑器中创建可出售的物品，并设置它们的价格和数量。学完本节后，你将能够让自己的商店正常售卖商品，让玩家用金币购买装备和道具。

### 操作步骤

1. **第一步：创建物品对象** — 在编辑器左侧的"游戏配置"面板中，找到"物品"（Item）选项并点击右键，选择"新建物品"[^1]。这样做会创建一个全新的物品模板，我们之后可以让商店出售它。

2. **第二步：设置物品属性** — 选中刚创建的物品，在右侧属性面板中可以修改它的名称、图标、模型等基本属性[^3]。建议把物品名称改成有意义的名字，方便后续管理。

3. **第三步：设置物品出售价格** — 在物品属性中找到"gold cost"（金币价格）字段，填入数字。这个数字决定了玩家购买该物品需要花费多少金币[^1]。

4. **第四步：设置物品分类** — 在物品属性中设置"abel"（标签）或"classification"（分类），这影响物品在商店界面中显示的位置。常见分类包括：永久使用、消耗品、武器、防具等。

5. **第五步：设置出售数量** — 在商店对象的属性中，找到"库存Maximum"（最大库存）字段。这个数字限制玩家总共能购买多少个该物品[^2]。如果填0，则表示无限供应。

6. **第六步：添加物品到商店** — 打开商店对象，在"Items Sold"（出售物品）列表中添加你刚创建的物品。保存地图后，玩家就能在商店中看到这个商品了。

### 小结

完成以上步骤后，你的商店应该能正常显示物品，玩家可以用金币购买了。建议先测试一下购买功能是否正常——如果玩家看不到物品或无法购买，很可能是物品没有正确添加到商店的出售列表中。

## 配置金币系统

在本节中，你将学会如何在地图中设置金币系统，包括玩家开始游戏时有多少金币、怎么获得更多金币、以及如何在界面上显示金币数量。完成这些设置后，你的商店系统就能正常运作了！

### 操作步骤

1. **第一步：创建金币变量** — 在触发器编辑器中，我们需要用"变量"来存储金币数据。点击触发器编辑器左上角的"变量"按钮（一个绿色加号），在弹出的列表中找到"整数"类型，新建一个变量并命名为 `玩家金币`。[^1]

2. **第二步：设置玩家初始金币** — 新建一个"初始化"触发器，在"事件"中选择"地图初始化"。然后添加动作：设置变量值 → 选择`玩家金币` → 设为初始金币数量（比如 100）。这样玩家进入游戏时就会自动获得设定的金币。[^3]

3. **第三步：创建获得金币的触发器** — 如果你想让玩家通过击杀怪物获得金币，需要新建触发器。事件设置为"单位死亡"，条件设置为"死亡单位是怪物"，动作设置为"SetPlayerState（设置玩家状态）→ 金币资源 + 10"。这样每次击杀怪物就能获得金币了。[^1]

4. **第四步：设置金币上限** — 在同一个获得金币的触发器中，添加条件检查：`如果 玩家金币 小于 99999`，再执行增加金币的动作。这样可以防止金币无限增长超过上限。

5. **第五步：在界面上显示金币** — 按 `F4` 打开对话框编辑器，选择"资源标签"，将其拖到屏幕右上角。右键设置其"资源类型"为"金币"，关联到`玩家金币`变量。这样玩家就能随时看到自己有多少钱了。[^2]

> **💡 新手提示**：变量名最好用英文或拼音，避免使用中文变量名，这在后续写复杂触发器时可以避免很多奇怪的报错。

> **⚠️ 常见错误**：很多新手忘记给"单位死亡"触发器设置"条件"，导致所有单位死亡都会给金币，包括玩家自己的单位！解决方法是在条件栏加上 `死亡单位 的 所属玩家 != 玩家1（红色玩家）`。

### 小结

完成以上步骤后，你的地图应该已经具备完整的金币系统了：玩家开始时有初始金币，击杀怪物可以获得更多，同时屏幕上会实时显示当前金币数量。下一步你可以开始制作商店，让玩家真正开始"花钱"了！

## 使用触发器完善商店功能

在本节中，你将学习如何使用**触发器**（Trigger）让商店真正能够运作起来。触发器是什么？简单来说，触发器就是一段"如果...就..."的规则。比如"如果玩家点击了购买按钮，就扣除他的金币并给他物品"。没有触发器，你的商店界面再漂亮也卖不出任何东西[^1]。

### 商店购买事件触发器

1. **打开触发编辑器** — 在编辑器顶部菜单栏找到"触发器"按钮（图标像一个闪电⚡），点击进入触发编辑器界面[^3]
2. **创建新触发器** — 在左侧面板右键点击，选择"新建触发器"，给它起个名字比如"商店购买"
3. **添加事件** — 事件就是"什么时候发生"。点击事件区域，选择"单位 - 单位进入区域"，然后用编辑器左边的"区域工具"画一个覆盖商店的区域
4. **添加条件** — 条件就是"要不要执行"。右键点击"条件"，添加"玩家 - 玩家属性"检查，比如"检查玩家金币是否大于等于物品价格"
5. **添加操作** — 操作就是"具体做什么"。在操作区域添加"玩家 - 扣除金币"和"物品 - 创建物品给单位"

> **⚠️ 常见错误**：很多新手只添加了事件和操作，忘记添加条件。结果就是玩家金币不足时也能"购买"，物品会凭空出现！一定要记得加条件检查。

### 购买成功后执行的操作

购买成功的流程需要按顺序执行以下操作：

1. **扣除玩家金币** — 在触发器的操作中添加"玩家 - 设置玩家属性"，将金币减少对应物品价格
2. **给予玩家物品** — 使用"英雄 - 给予物品"操作，让物品直接进入英雄背包[^1]
3. **播放音效或显示提示** — 添加"游戏 - 显示普通文本消息"告诉玩家购买成功，比如"购买成功！"
4. **记录购买日志** — 可以用变量记录购买次数，方便后续做统计或成就系统

> **💡 新手提示**：如果英雄背包满了，物品会给不出来！记得在"给予物品"后面加一个判断，检查背包是否有空间。

### 购买失败的情况处理

当玩家金币不足时，我们需要友好地告诉他原因：

1. 在触发器的"条件"里，添加另一个分支——右键条件选择"否则"（如果条件不满足就执行这个）
2. 在"否则"下面添加"游戏 - 显示普通文本消息"，内容写"金币不足！还需要XX金币"
3. 可以额外播放一个"失败音效"让玩家知道交易没成功

> **💡 新手提示**：显示的文本最好用不同颜色区分成功和失败，比如绿色表示成功，红色表示失败。在文本前后加`|cffff0000`（红色）和`|r`可以改变颜色。

### 小结

完成以上步骤后，你的商店就已经具备完整的购买功能了！玩家走到商店区域，点击购买按钮，系统会自动检查金币是否足够，然后扣除金币并给予物品。如果金币不足，会显示友好的提示消息。触发器的核心逻辑就是：**事件 → 条件判断 → 满足则执行成功操作 / 不满足则执行失败提示**。你可以复制这个触发器，修改物品类型和价格，就能快速制作多个不同的商店！

## 常见问题与调试

在制作商店系统的过程中，新手经常遇到各种问题。本节将帮你排查最常见的三个问题：**商店不显示物品**、**玩家无法购买**、**金币扣除不正确**。学完以后，你就能自己解决这些小毛病，让商店正常工作。

### 商店不显示物品

1. **检查商店单位是否正确放置** — 在地图编辑器左侧的「单位」面板中，找到「商店」类别的单位（如「背包商店」），将它拖到你的地图上[^3]。确保商店单位放在可见的区域内。
2. **检查商店的物品列表** — 选中商店单位，按 **F4** 打开对象编辑器（用来修改单位属性的面板），在「命令」栏中确认物品已添加[^1]。如果列表是空的，点击「物品」旁边的 **+** 号手动添加。
3. **检查玩家的视野** — 商店必须放在玩家视角可以看到的区域，且没有被地形遮挡。

> **⚠️ 常见错误**：很多新手把商店放到地图边缘或者水下/悬崖下面，导致玩家根本找不到商店。解决方法：把商店拖到主基地区域附近。

### 玩家无法购买物品

1. **检查触发器（Trigger）是否启用** — 打开触发器编辑器（左上角「触发器」按钮），找到负责「玩家点击商店」或「购买物品」的触发器，确认它前面没有红色的「X」标记（表示已禁用）[^1]。
2. **检查触发器条件** — 触发器中可能有「玩家金币 >= 物品价格」的条件[^2]。如果条件写错了，玩家即使有钱也买不了。检查条件语句是否正确匹配了你的金币系统。
3. **测试金币系统是否启动** — 给玩家添加一些初始金币（在触发器中设置「设置玩家金币」动作），确保金币系统已经运行。

> **💡 新手提示**：触发器就像是「自动化的机器人脚本」——你告诉它「当玩家点击物品时，就执行XXX动作」。如果触发器没开，它就不会工作。

### 金币扣除不正确

1. **检查价格设置** — 选中商店单位，按 **F4** 打开对象编辑器，找到物品的价格设置[^1]。确认「成本」数值是你想要的数字。
2. **检查金币扣除触发器** — 在购买物品的触发器中，找到「扣除玩家金币」动作，确认它使用的是「物品价格」变量或直接写对了数值[^2]。
3. **检查是否有重复扣除** — 如果同一个购买动作触发了两次，金币会被扣两次。检查触发器中没有重复的「扣除金币」动作。

> **⚠️ 常见错误**：新手常常把「物品实际价格」和「玩家需要支付的价格」混淆。比如物品成本是100，但触发器里写的是200，就会出现扣钱不对的情况。

### 小结

完成以上排查后，你的商店系统应该能正常显示物品、允许玩家购买、并正确扣除金币。建议用「测试地图」功能（按住空格键点击「测试」）多试几次购买流程，确保各种情况下都正常。如果还有问题，可以打开触发器的「注释」功能给自己写备注，方便追踪逻辑。
